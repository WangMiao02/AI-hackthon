# beauty_service — 美容预约

## 1. 执行入口判断

**首先判断触发来源，跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时系统触发（周期复约） | `trigger_type == "scheduled"` + 周期提醒任务存在 | → §5 周期复约提醒 |
| 用户主动唤起 | 对话中含美容/护肤/脱毛/SPA等意图 | → §2 意图识别与分流 |
| 用户未完成配置 | 首次使用，常用地址未设置 | → §6 Onboarding |

---

## 2. 意图识别与分流

| 用户意图 | 示例表达 | 服务类型 | 跳转 |
|---------|---------|---------|------|
| 面部护理 | "做个脸""面部补水""清洁毛孔" | `facial` | → §3 服务查询 |
| 皮肤管理 | "皮肤管理""小气泡""刷酸" | `skin_care` | → §3 服务查询 |
| SPA/身体护理 | "做个SPA""身体护理""精油按摩" | `spa_body` | → §3 服务查询 |
| 光子脱毛 | "脱毛""腋下脱毛""冰点脱毛" | `hair_removal` | → §3 服务查询 |
| 泛护肤意图 | "护肤""想保养一下" | — | 追问：「想做面部护理、皮肤管理、SPA 还是脱毛？」 |

> **脱毛类特殊处理**：意图识别为 `hair_removal` 后，追问「脱毛部位是？（腋下/手臂/腿部/唇部/全身）」，将部位作为附加筛选条件传入查询接口。

---

## 3. 服务查询与推荐

**输入**：服务类型 + 用户地址（GPS/常用地址）+ 脱毛部位（如有）

**执行步骤**：

1. 调用美团丽人接口，查询附近门店及对应项目
2. 按以下规则排序，取前 **2 家** 展示，超出折叠：

   | 优先级 | 筛选维度 | 说明 |
   |--------|---------|------|
   | 1 | 历史消费过该门店 | 熟悉环境，用户信任度高 |
   | 2 | 评分 | 4.8分及以上优先 |
   | 3 | 距离 | 同等条件取距离最近 |
   | 4 | 价格 | 同等条件取性价比最高 |

   **冷启动处理**（无历史消费记录）：按综合评分从高到低排列，展示评分 + 服务次数 + 人均价格

3. 无可用门店 → 提示「附近暂无匹配的美容门店，请尝试调整位置或项目类型」
4. 有结果 → 跳转 §4 预约确认

---

## 4. 预约确认

**执行步骤**：

1. 展示门店推荐卡片（含评分、项目价格、距离）
2. 用户选择门店 → 展示该店可选项目列表
3. 用户选择项目 → 展示可选技师（含评分、擅长项目）
4. 用户选择技师（或跳过由系统分配）→ 展示可选时段
5. 用户选择时段 → 展示预约确认卡片
6. 用户确认 → 调用 `order` skill 完成支付 → §4.1 预约结果处理

卡片格式及时段交互详见 [references/booking-flow.md](references/booking-flow.md)

### 4.1 预约结果处理

**成功**：
```
✅ 预约成功！
💆 [项目名称] · [门店名称]
👤 技师：[技师姓名] ⭐[评分]
📅 [预约日期] [时间段]
📍 [门店地址]
💰 ¥[金额]
📌 已写入今日行程，明早早报会提醒你出发时间。
```
→ 将预约信息写入当日行程，供 `morning_briefing` 早报「今日行程」模块读取，统一处理叫车提醒
→ 预约成功卡片底部追加轻量收尾文案：「下次想做美容，随时来找我预约～」，不强制询问周期；周期复约设置入口保留在 §5，用户主动说「帮我设置提醒」时才触发

**失败**：
→ 提示失败原因，保留预约信息，引导重试或换门店/时段

## 5. 周期复约提醒

### 5.1 首次设置（用户主动触发）

用户说「帮我设置美容提醒」「下次提前提醒我」等时触发，询问：

```
🔄 好的！多久提醒你一次？

[2周后]  [1个月后]  [自定义]
```

用户选择周期后，记录以下信息用于下次复约：
- 服务类型（`facial` / `skin_care` / `spa_body` / `hair_removal`）
- 上次门店信息（优先复用）
- 上次技师信息（优先复用）
- 上次项目信息
- 上次服务地址
- 下次提醒时间 = 当前日期 + 周期 - 1天

### 5.2 到期提醒推送（定时触发）

到期前一天推送复约卡片，格式及交互详见 [references/periodic-reminder.md](references/periodic-reminder.md)

## 7. 边界处理速查

| 场景 | 处理 |
|------|------|
| 附近无可用门店 | 提示调整位置或扩大搜索范围 |
| 用户选择时段已被预约 | 提示「该时段刚被预约」，引导重新选择 |
| 支付失败 | 提示原因，保留预约信息可重试 |
| 门店临时闭店 | 推送通知，引导重新选择门店 |
| 常用地址未设置 | 触发 §6 Onboarding |
| GPS 未授权 | 降级使用 Onboarding 中的常用地址 |
| 用户问到医美项目 | 明确告知「水光针/玻尿酸等医美项目暂不支持预约，建议前往正规医美机构咨询」 |
| 周期提醒到期但用户已手动预约 | 检测到近期已有预约记录则静默跳过 |
| 脱毛类无法确定部位 | 展示全部脱毛项目列表供选择 |

- 退款/改约流程
