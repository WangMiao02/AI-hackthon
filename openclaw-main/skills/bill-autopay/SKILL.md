# bill_autopay — 缴费管家

## 0. Demo 模式声明

本 Skill 运行于 Demo 模式。账单数据来源于本地 mock 数据库（`references/mock-bills.json`），
模拟生活缴费聚合平台返回结果。数据结构覆盖水电燃气、通讯、物业、信用卡等主要品类。
支付环节模拟成功但不触发真实扣款。

**接口模拟方式**：
```
GET /api/bills?user_id=demo_user
Response: references/mock-bills.json

POST /api/bills/pay
Request: { "bill_id": "xxx", "amount": 123.45 }
Response: { "status": "success", "transaction_id": "TXN..." }
```

---

## 1. 执行入口判断

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时触发（缴费日前1天） | `trigger_type == "due_reminder"` | -> S4 缴费提醒推送 |
| 定时触发（月度汇总） | `trigger_type == "monthly_summary"` | -> S5 月度缴费汇总 |
| 定时触发（异常检测） | `trigger_type == "anomaly_check"` | -> S6 异常金额预警 |
| 用户主动查询 | 对话中含缴费/水电/话费/欠费等意图 | -> S2 前置数据拉取 |
| 用户首次使用 | 无账单配置记录 | -> S7 Onboarding |

---

## 2. 前置数据拉取

**执行步骤**：

1. 调用账单聚合接口：`GET /api/bills?user_id=demo_user`
2. 解析返回数据，按紧急程度排序：
   - `overdue`（已逾期）— 最紧急
   - `due_today`（今日到期）— 紧急
   - `due_soon`（3天内到期）— 需关注
   - `paid`（已缴）— 信息类
3. 数据就绪 -> 跳转 S3 展示

## 4. 缴费提醒推送

> 定时触发：每项账单截止日前1天 9:00 推送。

```
┌──────────────────────────────────────┐
│ 💡 明天有账单到期                     │
│                                      │
│ [账单名称]                           │
│ 账户：[账户信息]                      │
│ 应缴金额：¥[金额]                    │
│ 截止日期：明天（[具体日期]）           │
│                                      │
│ 对比上月：[金额变化说明]              │
│                                      │
│    [帮我缴]   [我自己缴]   [已缴过了] │
└──────────────────────────────────────┘
```

**用户响应分支**：

| 操作 | 处理 |
|------|------|
| 帮我缴 | 调用缴费执行接口，返回缴费结果 -> S4.1 |
| 我自己缴 | 标记为"用户自行处理"，截止日当天再确认一次 |
| 已缴过了 | 标记为已缴，更新状态，不再提醒 |

### 4.1 缴费执行结果

**成功**：
```
缴费成功 ✓
[账单名称] ¥[金额]
交易单号：[TXN编号]
扣款方式：[支付方式]
```
-> 同步记录至 budget-manager

**失败**：
```
缴费未成功，可能原因：[余额不足/网络异常/账户信息有误]
建议：[具体建议]
需要我稍后重试吗？
```

## 6. 异常金额预警

> 定时触发：新账单出来时，与过去3个月均值对比，偏差超过 30% 则预警。

**触发条件**：
- 新账单金额 > 近3月均值 * 1.3（偏高）
- 新账单金额 < 近3月均值 * 0.5（异常偏低，可能是抄表错误）

**推送内容**：
```
⚠️ [账单名称] 金额异常

本期：¥[金额]
近3月平均：¥[均值]
偏差：+[X]%

可能原因：
· 夏季空调用电增加
· 抄表周期不一致
· 确认是否有设备异常耗电

操作：[正常，帮我缴] [我核实一下] [联系客服]
```

## 7. Onboarding（首次配置）

引导用户添加需要追踪的账单项目：

```
帮你追踪生活缴费，不再忘缴欠费～
先告诉我你需要管理哪些账单？

常见类型：
1. 电费（国家电网/南方电网）
2. 水费
3. 燃气费
4. 手机话费
5. 宽带费
6. 物业费
7. 停车费
8. 信用卡还款

选几个你需要的，或者直接说"全都要"
```

每项需要的信息：

| 配置项 | 示例 |
|--------|------|
| 账单类型 | 电费 |
| 账户标识 | 户号 1234567890 |
| 缴费周期 | 每月 |
| 通常截止日 | 每月25号 |
| 历史平均金额 | 约200元/月 |
| 是否自动扣款 | 否（需手动缴） |

> 已开通自动扣款的账单，Skill 仅做"确认是否正常扣款"的监控，不重复操作。

## 8. 边界处理速查

| 场景 | 处理 |
|------|------|
| 用户未配置任何账单 | 触发 Onboarding 引导 |
| 已开通自动扣款的账单 | 仅监控扣款是否成功，不主动代缴 |
| 账单金额为0 | 跳过，不推送（可能是抄表周期空月） |
| 用户说"我已经交了" | 标记已缴，不再提醒 |
| 信用卡最低还款 vs 全额 | 展示两个金额供选择，默认建议全额 |
| 逾期产生滞纳金 | 提示滞纳金金额，建议尽快缴纳 |
| 用户问贷款/房贷 | 告知"贷款和分期由预算管家追踪，我负责日常生活缴费" |
| 缴费平台维护中 | 提示稍后重试，设置2小时后自动重试 |
