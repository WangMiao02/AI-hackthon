# 超支预警 — 卡片模板与触发机制

## 文案基调原则

提醒语气轻松，像朋友随口说一句，不制造焦虑感。避免"超支！""预算已超！"等强烈措辞，
改为"花得差不多了""这块花多了一点"等平和表达。目的是让用户知情，而不是让用户紧张。

---

## 预警节点与去重规则

- 用户可在 Onboarding 步骤二自定义多个预警节点（如 50% / 70% / 100%），每个节点独立计算
- **每个节点当月只推一次**，推送后写入该节点的已推标记，之后静默，不重复提醒
- 类目预警与总预算预警独立去重，互不影响
- **退款恢复**：如果退款后消费降回某节点阈值以下，清除该节点的已推标记，下次再次到达时重新推送，并附带一句退款恢复提示（见 §退款恢复文案）

---

## 推送顺序

同一笔消费同时触发分类目预警和总预算预警时：**先推分类目预警，再推总预算预警**，两张卡片依次展示，不合并。

---

## 分类目预警卡片

### 接近上限（消费达该类目预算的用户设定阈值）

```
┌──────────────────────────────────────┐
│ 🍱 外卖这块花得差不多了              │
│                                      │
│ 本月外卖 ¥1,350 / 预算 ¥1,500       │
│ 还剩 ¥150，剩 12 天                  │
│                                      │
│ [知道了]  [看看明细]  [调整预算]     │
└──────────────────────────────────────┘
```

### 已超出类目预算

```
┌──────────────────────────────────────┐
│ 🍱 外卖这个月多花了一点              │
│                                      │
│ 本月外卖 ¥1,680 / 预算 ¥1,500       │
│ 多了 ¥180，后面注意一下就好          │
│                                      │
│ [知道了]  [看看明细]  [调整预算]     │
└──────────────────────────────────────┘
```

**用户响应分支：**

| 操作 | 处理 |
|------|------|
| 点击「知道了」 | 写入该类目当月已推标记，之后静默 |
| 点击「看看明细」 | 加载 `references/weekly-report.md` §消费看板（筛选该类目） |
| 点击「调整预算」 | 加载 `references/onboarding.md` §预算调整 |

## 总预算预警卡片

### 达到用户设定的某个节点阈值（如 70%）

```
┌──────────────────────────────────────┐
│ 💰 这个月花了七成了                  │
│                                      │
│ 已花 ¥2,275 / 预算 ¥3,250           │
│ 还剩 ¥975，剩 12 天，日均约 ¥81     │
│                                      │
│ [知道了]  [看看花哪了]  [调整预算]   │
└──────────────────────────────────────┘
```

### 达到 100%（已用完预算）

```
┌──────────────────────────────────────┐
│ 💰 这个月预算用完啦                  │
│                                      │
│ 已花 ¥3,280 / 预算 ¥3,250           │
│ 多了 ¥30，还有 12 天，量力而行就好  │
│                                      │
│ [知道了]  [看看花哪了]  [调整预算]   │
└──────────────────────────────────────┘
```

> 文案中的"几成"根据实际阈值动态生成：50% → 一半、60% → 六成、70% → 七成、80% → 八成、90% → 九成、100% → 用完啦。

**用户响应分支：**

| 操作 | 处理 |
|------|------|
| 点击「知道了」 | 写入该节点当月已推标记，之后静默 |
| 点击「看看花哪了」 | 加载 `references/monthly-report.md` 消费分析 |
| 点击「调整预算」 | 加载 `references/onboarding.md` §预算调整 |

---

## 退款恢复文案

退款后，若消费从某节点阈值以上降回阈值以下，清除该节点已推标记，并静默附加一条提示：

```
✅ 刚才那笔退款已更新，本月消费回到 ¥2,180（67%），
   之前的预算提醒也跟着撤了～
```

> 仅在退款导致阈值穿越时展示，普通退款（未穿越阈值）静默处理。
> 若用户偏好为静默入账，则此条提示也静默，不展示。
