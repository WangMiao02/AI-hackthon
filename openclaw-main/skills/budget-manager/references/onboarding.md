# Onboarding 与预算调整 — 配置流程与卡片模板

## 触发条件

- 首次使用，预算配置未设置
- 用户点击任意「调整预算」按钮
- 用户主动表达"设置预算""把外卖预算改成 XX"等

---

## 首次配置（Onboarding）

### 文案原则

开场语气轻松自然，像朋友说话，不出现"检测到""系统""首次使用"等机器感措辞。

### 步骤一：一屏完成分类预算设置

直接展示一张卡片，包含吃/行/住/玩四大类，每类预置默认勾选档位（基于城市平均消费数据），用户可点击切换档位，也可在输入框里直接改金额，底部实时汇总总预算。其余小类（医药、日用品、其他）按比例自动估算，附在汇总行下方。

│    [¥800以内]  [¥800–1500 ●]  [¥1500+]     │
│    金额：[¥1,200          ]                  │
│                                              │

│    金额：[¥500            ]                  │
│                                              │

│    [基本不住]  [偶尔 ●]        [比较频繁]   │

**交互细节：**

- 默认勾选档位基于用户所在城市同龄人平均消费数据，冷启动时使用全国平均值
- 点击其他档位 → 该档位高亮，输入框金额自动更新为该档对应预设值
- 直接修改输入框金额 → 档位自动切换到最接近的档，或三档均取消高亮（自定义状态）
- 月总预算实时计算：四大类金额之和 + 估算小类金额
- 估算小类固定为四大类合计的 32%（医药 12% + 日用品 12% + 其他 8%），用户保存后可在预算调整里单独修改

### 步骤二：提醒偏好设置

预算卡片保存后立即展示步骤二，三项配置一屏完成，默认帮用户勾好，可直接点「好了，开始吧」跳过细调。

```
┌──────────────────────────────────────────────┐
│ 最后设置一下提醒偏好，之后随时可改           │
│                                              │
│ 🔔 花到多少的时候提醒我一下？                │
│    可以选多个节点，每个只提醒一次            │
│                                              │
│    [ ] 50%   [✓] 70%   [✓] 100%            │
│    [ ] 自定义：[      %]  [+ 添加]          │
│                                              │
│ 📊 定期给我发消费报告                        │
│    [不用]  [每天]  [每周 ●]                 │
│    每周的：[周一 ●][周二][周三][周四][周五]  │
│    几点推：[09:00          ]                 │
│                                              │
│ ✅ 每次下单后告诉我已记账                    │
│    [每次都说]  [不用，静默就好 ●]           │
│                                              │
│              [好了，开始吧]                  │
└──────────────────────────────────────────────┘
```

**交互细节：**

- 默认勾选 70% 和 100% 两个节点，用户可勾选/取消任意节点，也可点「+ 添加」输入自定义百分比（1–99 整数）
- 每个节点独立去重：当月该节点触发推送后写入已推标记，之后静默；若退款后消费降回该节点以下，清除标记并推送退款恢复提示
- 报告频率选「每周」后，展开「周几」和「几点」两个选项：周几默认周一，几点默认 09:00，均可修改；选「每天」则只需设定推送时间
- 入账提醒偏好：选「每次都说」→ 每笔入账后展示确认；选「不用，静默就好」→ 全程静默，退款恢复提示也同步静默

**用户响应分支：**

| 操作 | 处理 |
|------|------|
| 点击「好了，开始吧」 | 保存偏好配置，进入预算余额速查（SKILL.md §2.1） |
| 修改某项后点「好了，开始吧」 | 以修改后的值保存 |

## 预算调整

保存后随时可调，支持两种方式：

### 自然语言调整

| 示例表达 | 处理逻辑 |
|---------|---------|
| "把外卖预算改成 2000" | 直接修改对应类目输入框，重新汇总 |
| "总预算加 1000" | 按当前各类目比例分配增量 |
| "外卖和打车的预算对换一下" | 交换两个类目金额 |
| "加一个 80% 的提醒" | 在预警节点列表中添加 80% 节点 |
| "把 100% 的提醒去掉" | 移除 100% 节点 |
| "关掉每周报告" | 将报告周期改为「不用」 |
| "改成周五推" | 将周报推送日改为周五，时间不变 |
| "改成早上 8 点推" | 将推送时间改为 08:00，周几不变 |
| "改成每次下单都告诉我" | 将入账提醒改为「每次都说」 |

### 交互式调整
