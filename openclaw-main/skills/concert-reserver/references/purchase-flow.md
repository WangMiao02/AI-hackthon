# 抢票与购票 — 卡片模板与流程细节

## 抢票模式（演唱会/热门演出）

**前置条件**：用户已确认抢票配置（票档、数量、实名人）

执行步骤：
1. 开售时刻（精确到秒）自动调用猫眼抢票接口
2. 按优先级尝试：目标票档 → 备选票档 → 同价位其他区域

### 抢票成功卡片

```
┌──────────────────────────────────────┐
│ 🎉 抢票成功！                        │
│                                      │
│ 🎤 周杰伦「嘉年华」北京站            │
│ 📅 6月15日（周日）20:00              │
│ 💺 内场 B3区 12排 15-16号            │
│ 💰 ¥1280 × 2 = ¥2,560.00            │
│ 👤 张思怡、李明                      │
│                                      │
│ ⏱ 请在 15 分钟内完成支付             │
│                                      │
│      [立即支付]     [取消订单]       │
└──────────────────────────────────────┘
```

→ 点击「立即支付」→ 调用 `order` skill 完成支付
→ 超时未支付 → 推送支付催促提醒


→ 选择其他票档 → 进入常规购票
→ 点击「帮我蹲票」→ 进入候补蹲票

```
┌──────────────────────────────────────┐
│ 🎫 确认购票                          │

│ 💰 合计：¥760.00                     │
│ 👤 实名人：张思怡、李明              │
│ 支付方式：美团支付（默认）           │
│                                      │
│      [确认购票]     [换场次]         │
└──────────────────────────────────────┘
```

**用户响应分支：**

| 操作 | 处理 |
|------|------|
| 点击「确认购票」 | 调用 `order` skill 完成支付 → 购票结果处理 |
| 点击「换场次」 | 返回 §3 重新选择 |

## 候补蹲票（后台静默监控）

```
┌──────────────────────────────────────┐
│ 👀 已开启蹲票模式                    │
│                                      │
│ 🎤 周杰伦「嘉年华」北京站            │
│ 目标票档：内场 ¥1280 / 看台 ¥880    │
│ 数量：2 张                           │
│                                      │
│ 📌 有退票释放时我会第一时间通知你    │
│ ⏱ 蹲票有效期：演出前 3 天自动结束   │
│                                      │
│         [好的]     [取消蹲票]        │
└──────────────────────────────────────┘
```

→ 监控到退票释放 → 推送通知，用户确认后立即下单
→ 蹲票超时（演出前 3 天）→ 推送「蹲票已结束，未检测到余票释放」

```
✅ 购票成功！

🎤 周杰伦「嘉年华」北京站
📅 6月15日（周日）20:00
💺 内场 B3区 12排 15-16号 × 2张

📌 演出前一天我会提醒你出发时间和注意事项。
👤 请凭身份证原件入场，不可转赠。
```

→ 设置演出前一天提醒任务
→ 如演出在外地 → 推送联动推荐：

```
📌 这场演出在上海，要帮你安排出行吗？

[帮我订票去上海] [帮我订酒店] [不用了]
```

→ 点击「帮我订票去上海」→ 触发 `trip_reserver`
→ 点击「帮我订酒店」→ 触发 `hotel_reserver`
