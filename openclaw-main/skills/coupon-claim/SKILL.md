# coupon_claim — 一键领券

## 1. 执行入口判断

**首先判断触发来源，跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时系统触发 | `trigger_type == "scheduled"` (每日 10:00) | → §2 筛选流程 |
| 用户主动唤起 | 对话中含券/优惠/领券等意图 | → §2 筛选流程 |

> 本 Skill 无强依赖的 Onboarding 配置，用户可选择性屏蔽不感兴趣的券种；未配置时由历史消费数据自动过滤。

---

## 2. 券筛选与排序

**输入**：用户历史消费记录 + 当日可领券池

**筛选规则（先过滤，再排序）**：

过滤掉：
- 与用户历史消费**无类目关联**的券（如用户从不使用丽人服务 → 过滤丽人券）
- 用户**已领取未使用**的同类券（避免重复推送）
- 用户在设置中**主动屏蔽**的券种

排序规则（综合倒序，力度权重 > 关联性权重）：

| 优先级 | 维度 | 说明 |
|--------|------|------|
| 1 | 券力度 | 优惠金额 / 折扣力度，绝对金额优先 |
| 2 | 消费关联性 | 与用户近期高频消费类目的匹配程度 |

示例：打车满50减15 > 酒旅9.5折 > 外卖满30减5（用户高频打车+酒旅，偶尔点外卖）

如果**无匹配券** → 推送友好提示，Skill 结束

如果**有匹配券** → 跳转 §3 推送卡片

---

## 3. 推送卡片与领取

### 3.1 领券卡片（输出格式）

默认展示前 3 张，超出折叠，用户可点击展开查看全部。

```
┌──────────────────────────────────────┐
│ 🎟 今日有 5 张券等你领               │
│                                      │
│ 🥇 打车券   满50减15                 │
│ 🥈 酒旅券   9.5折                    │
│ 🥉 外卖券   满30减5                  │
│    + 2张更多优惠 ∨                   │
│                                      │
│          [一键全领]                  │
└──────────────────────────────────────┘
```

**用户响应分支**：
- 点击 `一键全领` → 批量调用领券接口 → §3.2
- 展开折叠后发现券已抢完 → 实时标注「已抢完」，不可点击

### 3.2 领取结果处理

**全部成功**：
```
✅ 已成功领取 5 张券！

· 打车券   满50减15   有效期至 5月24日
· 酒旅券   9.5折       有效期至 5月31日
· 外卖券   满30减5    有效期至 5月20日 ⚠️ 3天后到期
· ...

📌 外卖券将于 3 天后到期，记得使用！
```

> 有效期 ≤ 3 天的券高亮标出，并在底部单独提示。

**部分 / 全部失败**：
```
⚠️ 本次领取部分失败：
· 打车券 已达领取上限
· 其余 4 张已成功领取

[重试失败项]
```
→ 用户点击 `重试失败项` → 重新调用失败券的领券接口

---

## 4. 边界处理速查

| 场景 | 处理 |
|------|------|
| 当日无匹配券 | 推送友好提示，不打扰用户 |
| 已领取同类券未使用 | 过滤，不重复推送 |
| 用户屏蔽了某类券 | 严格遵守设置，不推送对应类目 |
| 券领取接口失败（部分/全部） | 区分提示，提供重试入口 |
| 展开折叠后券已被抢完 | 实时标注「已抢完」，不可领取 |

---

## 5. 本期不做（Backlog）

> 以下功能**不在当前 Skill 范围内**，遇到请告知用户暂不支持：

- 单张券独立领取 / 取消某张不领
- 券到期前独立推送提醒（本期仅在领取时附带一次性提醒）
- 第三方平台券聚合
- 消费场景内主动推荐（如打车时提示「你有打车券可用」）
