# expense_reimbursement — 一键报销

## 1. 执行入口判断

**首先判断触发来源，跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时系统触发 | `trigger_type == "scheduled"` | → §3 消费过滤 |
| 用户主动唤起 | 对话中含报销/消费/开具等意图 | → §2 前置检查 |
| 用户未完成配置 | Onboarding 信息缺失 | → §5 Onboarding |

---

## 2. 前置检查（用户主动触发时执行）

1. 读取用户 Onboarding 配置（可报销类目、推送频率、报销单抬头）
2. 如果**配置缺失** → 跳转 §5 Onboarding，Skill 结束
3. 如果**配置完整** → 跳转 §3

---

## 3. 消费记录过滤

**输入**：用户授权的美团交易数据 + Onboarding 中勾选的可报销类目

**执行步骤**：
1. 读取上一统计周期内的消费记录
2. 按用户配置类目过滤（如：✅ 酒旅 ✅ 打车 ❌ 外卖）
3. 排除已报销记录
4. 如果**无可报销记录** → 推送友好提示，Skill 结束
5. 如果**有可报销记录** → 跳转 §4 推送报销卡片

---

## 4. 推送卡片与报销单生成

### 4.1 报销卡片（输出格式）

默认全选所有可报销条目，合计金额实时响应用户勾选变化。

```
┌─────────────────────────────────────┐
│ 💼 本周有 3 条消费记录可报销          │
│ 统计周期：5月10日 - 5月16日          │
│                                      │
│ ✅ 5月12日  打车   ¥  89.00          │
│ ✅ 5月13日  酒旅   ¥ 980.00          │
│ ✅ 5月15日  打车   ¥ 211.00          │
│                      合计：¥1,280.00 │
│                                      │
│      [一键开具]      [稍后提醒]      │
└─────────────────────────────────────┘
```

每条记录字段：勾选框 / 消费日期 / 类目 / 金额 / 订单号（折叠）

**用户响应分支**：
- 点击 `一键开具` + 有选中条目 → §4.2 生成报销单
- 点击 `一键开具` + 无选中条目 → 提示「未选择任何条目」，返回卡片
- 点击 `稍后提醒` → Skill 结束，下次推送周期继续

### 4.2 生成报销单

报销单字段：

| 字段 | 来源 |
|------|------|
| 报销人姓名 / 工号 | Onboarding 配置 |
| 报销日期 | 生成当日 |
| 消费明细列表 | 用户确认的勾选条目 |
| 合计金额 | 自动汇总 |
| 底部备注 | "由真探AI管家生成" |

生成后 App 内图片预览，提供两个操作：

```
      [完成]        [导出PDF]
```

- `完成` → 标记本批记录为已报销，Skill 结束
- `导出PDF` → 导出文件，标记已报销，Skill 结束

---

## 5. Onboarding（首次配置）

引导用户依次填写：

| 配置项 | 示例 |
|--------|------|
| 可报销消费类目 | ✅ 酒旅 ✅ 打车 ❌ 外卖 |
| 推送频率 | 每天 / 每周五 |
| 报销单抬头（姓名/工号） | 张思怡 / EMP-20241 |

> 配置完成后自动跳转 §3 执行。已配置用户可在「报销设置」中修改。

---

## 6. 边界处理速查

| 场景 | 处理 |
|------|------|
| 未完成 Onboarding | 触发 §5 引导，Skill 暂不执行 |
| 统计周期内无可报销记录 | 推送友好提示，不打扰用户 |
| 全部条目取消勾选后点击开具 | 提示「未选择任何条目」，返回卡片 |
| 同一记录已报销 | 过滤，不重复推送 |
| 消费数据读取失败 | 提示「数据获取失败，请稍后重试」 |

---

## 7. 本期不做（Backlog）

> 以下功能**不在当前 Skill 范围内**，遇到请告知用户暂不支持：

- 对接企业报销系统自动提交
- 线下发票 OCR 识别
- 多币种 / 境外差旅报销
- 报销审批状态追踪
