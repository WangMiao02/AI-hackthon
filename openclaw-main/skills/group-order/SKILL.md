# group_order — 外卖拼单

## 1. 执行入口判断

**首先判断触发来源，跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 收到拼单分享 | 检测到拼单链接/分享卡片 | → §2 加入拼单 |
| 用户主动发起拼单 | 对话中含"发起拼单/一起点/拼个单"等意图 | → §3 创建拼单 |
| 上游联动（起送价不足） | 外卖下单场景中检测到订单金额未达起送价，Agent 主动提示"要不要拼个单"；**用户明确确认后**才进入流程（Agent 不得自动发起） | → §3 创建拼单 |

---

## 2. 加入拼单（收到分享）

解析拼单链接 → 展示提醒卡片 → 用户选菜提交 → 等待发起人下单。

**详细流程与卡片格式**：参见 [references/join-flow.md](references/join-flow.md)

---

## 3. 创建拼单（用户主动发起）

获取定位 → 选餐厅 → 创建拼单 → 分享给他人 → 截止后汇总 → 确认下单（触发 skill_order）。

**详细流程与卡片格式**：参见 [references/create-flow.md](references/create-flow.md)

---

## 4. 边界处理速查

| 场景 | 处理 |
|------|------|
| 拼单链接已过期 | 提示「该拼单已截止」，Skill 结束 |
| 截止时间到仅有发起人一人 | 提示「暂无人加入，是否自己下单？」，确认则转 skill_order |
| 参与人未在截止前选品 | 汇总时仅包含已选品的人，未选品者不计入 |
| 餐厅打烊/不可下单 | 创建拼单时拦截提示，引导换一家 |
| GPS 未授权 | 降级使用常用地址 |
| 常用地址也未设置 | 提示用户设置地址后重试 |

---

## 5. 本期不做（Backlog）

> 以下功能**不在当前 Skill 范围内**，遇到请告知用户暂不支持：

- AA 分账 / 费用均摊计算与收款
- 拼单内催单（催某人快点选）
- 跨餐厅拼单（一次拼单只能选一家店）
- 拼单历史记录查询
- 非外卖场景的拼团（团购券等）
