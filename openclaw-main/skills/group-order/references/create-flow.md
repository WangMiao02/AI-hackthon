# 创建拼单 — 详细流程

## 3.1 选择餐厅

1. 获取用户定位（GPS 优先，降级常用地址）
2. 查询定位 5km 内支持拼单的餐厅
3. 按以下规则排序推荐：

**输入**：定位坐标 + 用户历史订单

**排序规则**：

| 优先级 | 维度 | 说明 |
|--------|------|------|
| 1 | 历史高频店铺 | 用户近期下单次数最多的餐厅优先 |
| 2 | 评分 | 同等条件下优先店铺评分更高的 |
| 3 | 配送时长 | 同等条件下优先配送更快的 |

**冷启动处理**（无历史订单）：
→ 推送 5km 内热门餐厅，按销量倒序排列

取前 **3 家** 展示，超出折叠；如**无可拼单餐厅** → 提示「附近暂无支持拼单的餐厅」，Skill 结束

推送餐厅选择卡片：

```
┌──────────────────────────────────────┐
│ 🍽 选一家餐厅发起拼单                │
│                                      │
│ 🥇 老乡鸡（静安寺店）               │
│    均价 ¥25 · 配送约 25 分钟         │
│ 🥈 夸父炸串                          │
│    均价 ¥32 · 配送约 30 分钟         │
│ 🥉 遇见小面                          │
│    均价 ¥28 · 配送约 35 分钟         │
│                                      │
│  [选第一家] [选第二家] [选第三家]    │
│         [暂时不拼了]                 │
└──────────────────────────────────────┘
```

**用户响应分支**：
- 点击 `选第X家` → §3.2 创建拼单并分享
- 点击 `暂时不拼了` → Skill 结束

---

## 3.2 创建拼单并分享

```
┌──────────────────────────────────────┐
│ 📢 拼单已创建！                      │
│                                      │
│ 餐厅：老乡鸡（静安寺店）            │
│ 截止时间：11:30（默认30分钟后）      │
│ 当前参与：仅你自己                   │
│                                      │
│  [分享给同事]  [我先选菜]  [设截止]  │
└──────────────────────────────────────┘
```

**用户响应分支**：
- 点击 `分享给同事` → 生成拼单分享链接/卡片，用户自行转发
- 点击 `我先选菜` → 进入 §2.2 菜品选择（发起人自己也要选菜，参见 join-flow.md §2.2）
- 点击 `设截止` → 允许修改截止时间

---

## 3.3 截止后汇总确认

到达截止时间或发起人手动截止后，推送汇总卡片给发起人：

```
┌──────────────────────────────────────┐
│ 📋 拼单汇总                          │
│ 老乡鸡（静安寺店）· 共 3 人          │
│                                      │
│ 👤 你：黄焖鸡米饭（大份）   ¥28.00  │
│ 👤 小王：香菇滑鸡饭         ¥24.00  │
│ 👤 小李：农家小炒肉         ¥22.00  │
│                                      │
│ 配送费：¥5.00（均摊 ¥1.67/人）       │
│ 合计：¥79.00                         │
│                                      │
│      [确认下单]     [再等等]         │
└──────────────────────────────────────┘
```

**用户响应分支**：
- 点击 `确认下单` → 传入合并订单信息，触发 `skill_order`，Skill 结束
- 点击 `再等等` → 延长截止时间 15 分钟
