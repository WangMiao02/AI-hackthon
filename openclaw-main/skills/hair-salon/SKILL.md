# hair_salon — 美发预约

## 1. 执行入口判断

**首先判断触发来源，跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时系统触发（周期复约） | `trigger_type == "scheduled"` + 周期提醒任务存在 | → §5 周期复约提醒 |
| 用户主动唤起 | 对话中含剪发/烫发/染发/美发等意图 | → §2 意图识别与分流 |
| 用户未完成配置 | 首次使用，常用地址未设置 | → §6 Onboarding |

---

## 2. 意图识别与分流

| 用户意图 | 示例表达 | 服务类型 | 跳转 |
|---------|---------|---------|------|
| 剪发 | "剪个头发""修一下发型""头发太长了" | `cut` | → §3 门店查询 |
| 烫发 | "烫个头""想做个卷发""冷烫/热烫" | `perm` | → §3 门店查询 |
| 染发 | "染个颜色""挑染""想换个发色" | `color` | → §3 门店查询 |
| 护理 | "做个发膜""头发修复""烫染后护理" | `treatment` | → §3 门店查询 |
| 组合项目 | "剪+染""烫+护理" | `combo` | → §3 门店查询（标注组合需求） |
| 意图不明确 | "做个头发""约个Tony" | — | 追问：「想剪发、烫发、染发还是护理？」 |

> **组合项目处理**：识别到多个服务类型时，将组合需求传入查询接口，筛选支持该组合的门店，并在时长和价格上合并展示。

---

## 3. 门店与发型师查询

**输入**：服务类型 + 用户地址（GPS/常用地址）

**执行步骤**：

1. 调用美团丽人接口，查询附近门店及发型师
2. 按以下规则排序，取前 **2 家** 展示，超出折叠：

   | 优先级 | 筛选维度 | 说明 |
   |--------|---------|------|
   | 1 | 历史消费过该门店 | 熟悉环境，用户信任度高 |
   | 2 | 历史指定发型师在该店 | 优先展示有固定发型师的门店 |
   | 3 | 评分 | 4.8分及以上优先 |
   | 4 | 距离 | 同等条件取距离最近 |

   **冷启动处理**（无历史消费记录）：按综合评分从高到低排列，展示评分 + 服务次数 + 人均价格

3. 无可用门店 → 提示「附近暂无匹配的美发门店，请尝试调整位置」
4. 有结果 → 跳转 §4 预约确认

---

## 4. 预约确认

**执行步骤**：

1. 展示门店推荐卡片（含评分、人均价格、距离）
2. 用户选择门店 → 展示该店发型师列表（**含作品图**，详见 references/booking-flow.md）
3. 用户选择发型师（或跳过由系统分配）→ 展示可选项目及价格
4. 用户选择项目 → 展示可选时段
5. 用户选择时段 → 展示预约确认卡片
6. 用户确认 → 调用 `order` skill 完成支付 → §4.1 预约结果处理

> **作品图加载规则**：从该发型师的评论区/作品集拉取最近 **3 张**代表作品图，优先展示与本次服务类型匹配的作品（如用户选染发则优先展示染发作品）。图片加载失败时降级展示文字描述「擅长：[标签]」。

卡片格式及作品图交互详见 [references/booking-flow.md](references/booking-flow.md)

### 4.1 预约结果处理

**成功**：
```
✅ 预约成功！
✂️ [项目名称] · [门店名称]
👤 发型师：[姓名] ⭐[评分]
📅 [预约日期] [时间段]
📍 [门店地址]
💰 ¥[金额]
📌 已写入今日行程，明早早报会提醒你出发时间。
```
→ 将预约信息写入当日行程，供 `morning_briefing` 早报「今日行程」模块读取，统一处理叫车提醒
→ 预约成功卡片底部追加轻量收尾文案：「下次想做头发，随时来找我预约～」，不强制询问周期；周期复约设置入口保留在 §5，用户主动说「帮我设置提醒」时才触发

**失败**：
→ 提示失败原因，保留预约信息，引导重试或换门店/时段

## 5. 周期复约提醒

### 5.1 首次设置（用户主动触发）

用户说「帮我设置美发提醒」「下次提前提醒我」等时触发，询问：

```
🔄 好的！多久提醒你一次？

[3周后]  [1个月后]  [2个月后]  [自定义]
```

用户选择周期后，记录以下信息用于下次复约：
- 服务类型（`cut` / `perm` / `color` / `treatment` / `combo`）
- 上次门店信息（优先复用）
- 上次发型师信息（**强优先复用**，美发有强固定发型师偏好）
- 上次项目信息
- 下次提醒时间 = 当前日期 + 周期 - 1天

### 5.2 到期提醒推送（定时触发）

到期前一天推送复约卡片，格式及交互详见 [references/periodic-reminder.md](references/periodic-reminder.md)

## 7. 边界处理速查

| 场景 | 处理 |
|------|------|
| 附近无可用门店 | 提示调整位置或扩大搜索范围 |
| 用户选择时段已被预约 | 提示「该时段刚被预约」，引导重新选择 |
| 支付失败 | 提示原因，保留预约信息可重试 |
| 门店临时闭店 | 推送通知，引导重新选择门店 |
| 常用地址未设置 | 触发 §6 Onboarding |
| GPS 未授权 | 降级使用 Onboarding 中的常用地址 |
| 历史指定发型师已离职/不在该店 | 提示「上次的发型师已不在该店」，推荐同店其他发型师或换店 |
| 作品图加载失败 | 降级展示文字标签「擅长：剪发/烫发/染发」 |
| 组合项目该店不支持 | 提示「该店暂不支持组合预约，建议分次预约」 |
| 周期提醒到期但用户已手动预约 | 检测到近期已有预约记录则静默跳过 |
