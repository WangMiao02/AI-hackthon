# 周期复约提醒 — 卡片模板与交互细节

## 到期复约提醒卡片

到期前一天推送：

```
┌──────────────────────────────────────┐
│ 🔔 美发复约提醒                      │
│                                      │
│ 距离你上次「精剪」已过去 1 个月，    │
│ 头发该打理啦～                       │
│                                      │
│ 上次信息：                           │
│ 🏠 TONY'S HAIR STUDIO               │
│ 👤 Tony（店长）  ⭐ 4.9分            │
│ 💰 ¥128                              │
│                                      │
│ [一键复约（同门店同发型师）]         │
│ [换个门店]  [下次再说]  [取消提醒]   │
└──────────────────────────────────────┘
```

## 用户响应分支

| 操作 | 处理 |
|------|------|
| 点击「一键复约」 | 使用上次门店+发型师信息，直接跳转时段选择卡片（§4 时段选择） |
| 点击「换个门店」 | 跳转 §3 重新查询推荐门店 |
| 点击「下次再说」 | 将提醒推迟 3 天后再次推送（仅推迟一次，第二次仍不操作则本轮静默） |
| 点击「取消提醒」 | 删除该周期提醒任务，不再推送 |

## 发型师离职处理

复约时若检测到上次发型师已离职或不在该店：

```
┌──────────────────────────────────────┐
│ ⚠️ 上次的 Tony 老师已不在该店        │
│                                      │
│ 为你推荐同店其他发型师：             │
│ 👤 小K  ⭐ 4.8分 · 服务 632 次       │
│    作品：[图1] [图2] [图3]           │
│                                      │
│ [选小K]  [换个门店]  [取消]          │
└──────────────────────────────────────┘
```

## 推迟逻辑

- 第一次「下次再说」→ 3 天后再推送一次
- 第二次仍无操作 → 本轮静默，等待下一个完整周期后再提醒
- 用户主动预约后 → 以新预约日期重新计算下次提醒时间

## 复约信息更新

当用户通过复约完成新预约时，自动更新记录：
- 更新上次门店/发型师/项目信息（如有变化）
- 重新计算下次提醒时间 = 新预约日期 + 用户设定周期 - 1天
