# home_service — 家政服务预约

## 1. 执行入口判断

**首先判断触发来源，跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时系统触发（周期复约） | `trigger_type == "scheduled"` + 周期提醒任务存在 | → §5 周期复约提醒 |
| 用户主动唤起 | 对话中含保洁/维修/家电清洗等意图 | → §2 意图识别与分流 |
| 用户未完成配置 | 首次使用，常用地址未设置 | → §6 Onboarding |

---

## 2. 意图识别与分流

| 用户意图 | 示例表达 | 服务类型 | 跳转 |
|---------|---------|---------|------|
| 保洁 | "约个保洁""家里要打扫一下" | `cleaning` | → §3 服务查询 |
| 上门维修 | "马桶堵了""灯坏了""帮我找个师傅" | `repair` | → §3 服务查询（+故障描述） |
| 家电清洗 | "空调该洗了""洗衣机要清洗" | `appliance_clean` | → §3 服务查询 |
| 意图不明确 | "帮我约个家政" | — | 追问：「需要保洁、维修还是家电清洗？」 |

> **维修类特殊处理**：意图识别为 `repair` 后，追问一句「方便描述一下是什么问题吗？」，将故障描述作为附加信息传入查询接口，帮助匹配对应专项师傅。

---

## 3. 服务查询与推荐

**输入**：服务类型 + 用户地址（GPS/常用地址）+ 期望上门日期

**执行步骤**：

1. 调用美团家政接口，查询附近可用师傅
2. 按以下规则排序，取前 **2 条** 展示，超出折叠：

   | 优先级 | 筛选维度 | 说明 |
   |--------|---------|------|
   | 1 | 历史服务过该用户 | 熟悉环境，用户信任度高 |
   | 2 | 评分 | 4.8分及以上优先 |
   | 3 | 距离 | 同等条件取距离最近 |
   | 4 | 价格 | 同等条件取价格最低 |

   **冷启动处理**（无历史服务记录）：按评分从高到低排列，展示评分 + 服务次数

3. 无可用师傅 → 提示「附近暂无可用师傅，请尝试调整上门时间」
4. 有结果 → 跳转 §4 推送预约卡片

---

## 4. 预约确认

**执行步骤**：

1. 展示师傅推荐卡片（含评分、价格、可用时段）
2. 用户选择师傅 → 展示时间窗口选择
3. 用户选择时间 → 展示预约确认卡片
4. 用户确认 → 调用 `order` skill 完成支付 → §4.1 预约结果处理

卡片格式及时间窗口交互详见 [references/booking-flow.md](references/booking-flow.md)

### 4.1 预约结果处理

**成功**：
```
✅ 预约成功！
🧹 [服务类型] · [师傅姓名] ⭐[评分]
📅 [上门日期] [时间段]
📍 [服务地址]
💰 ¥[金额]
📌 上门前 30 分钟师傅会提前联系你。
```
→ 设置上门当天提前 1 小时的提醒任务
→ 推送周期复约询问：「下次需要我提前帮你预约吗？」，用户选择周期后写入提醒任务，详见 §5

**失败**：
→ 提示失败原因，保留预约信息，引导重试或换师傅

---

## 5. 周期复约提醒

### 5.1 首次设置（预约成功后触发）

预约成功卡片底部追加询问：

```
🔄 下次需要我提前帮你预约吗？

[7天后]  [14天后]  [1个月后]  [不用了]
```

用户选择周期后，记录以下信息用于下次复约：
- 服务类型（`cleaning` / `repair` / `appliance_clean`）
- 上次师傅信息（优先复用）
- 上次服务地址
- 上次时间窗口（作为默认选项）
- 下次提醒时间 = 当前日期 + 周期 - 1天

### 5.2 到期提醒推送（定时触发）

到期前一天推送复约卡片，格式及交互详见 [references/periodic-reminder.md](references/periodic-reminder.md)

## 6. Onboarding（首次配置）

| 配置项 | 示例 | 说明 |
|--------|------|------|
| 常用服务地址 | 北京市朝阳区XX路XX号XX室 | 默认上门地址 |
| 门锁方式 | 密码锁 / 需在家 / 物业代开 | 师傅上门前沟通依据 |
| 常用联系电话 | 138xxxxxxxx | 师傅联系方式 |

> 配置完成后自动跳转 §3 服务查询。已配置用户可在「家政设置」中修改。

---

## 7. 边界处理速查

| 场景 | 处理 |
|------|------|
| 附近无可用师傅 | 提示调整上门时间，或扩大搜索范围 |
| 用户选择时段已被预约 | 提示「该时段刚被预约」，引导重新选择 |
| 支付失败 | 提示原因，保留预约信息可重试 |
| 师傅临时取消 | 推送通知，自动匹配备选师傅或引导重新预约 |
| 常用地址未设置 | 触发 §6 Onboarding |
| GPS 未授权 | 降级使用 Onboarding 中的常用地址 |
| 维修类无法描述故障 | 跳过故障描述，展示通用维修师傅列表 |
| 周期提醒到期但用户已手动预约 | 检测到近期已有预约记录则静默跳过 |

## 8. 本期不做（Backlog）

- 月嫂 / 保姆 / 育儿嫂（重服务，需线下签约）
- 搬家服务
- 管道疏通
- 服务评价 / 投诉流程
- 师傅实时位置追踪
- 多地址管理（仅支持一个常用地址）
- 退款 / 改期流程
