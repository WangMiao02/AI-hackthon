# 周期复约提醒 — 逻辑与卡片模板

## 周期设置询问（预约成功后触发）

预约成功卡片推送完毕后，紧接着追加询问：

```
🔄 下次需要我提前帮你预约吗？

[7天后]  [14天后]  [1个月后]  [不用了]
```

**用户响应分支：**

| 操作 | 处理 |
|------|------|
| 选择「7天后」/「14天后」/「1个月后」 | 记录周期提醒任务（见下方数据结构），回复确认文案 |
| 点击「不用了」 | 不创建提醒任务，Skill 结束 |

**确认回复文案：**
```
✅ 好的，[X天/1个月]后我会提前一天提醒你预约。
```

**周期提醒任务数据结构：**

| 字段 | 内容 |
|------|------|
| `service_type` | 服务类型（cleaning / repair / appliance_clean） |
| `last_worker` | 上次师傅姓名 + ID（优先复用） |
| `address` | 上次服务地址 |
| `last_time_slot` | 上次时间窗口（如「09:00–11:00」，作为默认选项） |
| `remind_at` | 当前日期 + 周期天数 - 1天（提前一天提醒） |
| `period_days` | 7 / 14 / 30 |

## 到期复约提醒卡片（定时触发）

到期前一天，推送以下卡片：

```
┌──────────────────────────────────────┐
│ 🔄 该约保洁啦！                      │
│                                      │
│ 上次：王师傅 · 5月26日 09:00–11:00   │
│ 📍 朝阳区XX路XX号XX室                │
│                                      │
│ 王师傅明天可用时段：                 │
│   🟢 09:00–11:00（上次同一时段）     │
│   🟢 14:00–16:00                     │
│                                      │
│ [就这个时间]    [换个时间]           │
└──────────────────────────────────────┘
```

**字段说明：**

| 字段 | 说明 |
|------|------|
| 上次同一时段 | 优先展示与上次相同的时间窗口，标注「上次同一时段」 |
| 王师傅明天可用 | 若上次师傅明天有空则直接展示；若无空档则展示次优师傅 |

**用户响应分支：**

| 操作 | 处理 |
|------|------|
| 点击「就这个时间」 | 直接复用上次师傅 + 上次时间窗口 + 上次地址，跳转预约确认卡片，用户一次确认即完成预约 |
| 点击「换个时间」 | 复用上次师傅和地址，仅重新展示时间窗口选择卡片 |
| 不响应（超过 24 小时） | 静默跳过，不重复推送；下次周期照常计算 |

## 上次师傅不可用时的降级处理

若上次师傅在提醒日期前后 3 天均无可用时段：

```
┌──────────────────────────────────────┐
│ 🔄 该约保洁啦！                      │
│                                      │
│ 王师傅近期已约满，为你推荐：         │
│                                      │
│ 🥇 张师傅  ⭐ 4.9分  ¥68/小时        │
│    明天 09:00–11:00 可约             │
│                                      │
│ [预约张师傅]    [自己选]             │
└──────────────────────────────────────┘
```

→ 点击「预约张师傅」→ 跳转预约确认卡片（时间预填）
→ 点击「自己选」→ 跳转 §3 服务查询，重新走完整流程

---

## 周期提醒的更新与取消

用户可随时说：
- **「取消家政提醒」** → 删除当前周期提醒任务，回复 `✅ 已取消家政定期提醒`
- **「改成每月提醒」** → 更新 `period_days` 为 30，回复 `✅ 已更新为每月提醒`
- **「下次不用提醒了」**（在复约卡片上）→ 删除任务，不再推送
