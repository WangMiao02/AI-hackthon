# hotel_daily_report — 每日经营数据播报

## 1. 执行入口判断

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时系统触发 | `trigger_type == "scheduled"`（每日 08:00） | → §3 数据拉取 |
| 商家主动唤起 | 对话中含营收 / 入住率 / 数据 / 昨天怎么样等意图 | → §2 前置检查 |
| 上游 Skill 写入 | 各 Skill 推送日报数据（`trigger_type == "skill_write"`） | → 写入对应数据项，不触发播报 |
| 配置缺失 | 首次使用或配置不完整 | → §5 Onboarding |

---

## 2. 前置检查（商家主动触发时执行）

1. 读取商家配置，**配置缺失** → 跳转 §5 Onboarding，Skill 结束
2. **配置完整** → 跳转 §3 数据拉取

---

## 3. 数据拉取（静默）

### 3.1 平台数据接口读取

从榛果民宿 / 美团民宿商家后台接口读取：

| 数据项 | 口径说明 |
|--------|---------|
| 入住率 | 实际间夜数 ÷ 可售间夜数 |
| 营收 | 扣除平台佣金后的到手金额 |
| 间夜数 | 当日实际完成入住的间夜总数 |
| 平均每间夜收入 | 营收 ÷ 可售间夜数 |
| 好评数 | 当日新增 4-5 星评价数 |
| 差评数 | 当日新增 1-3 星评价数 |

### 3.2 各 Skill 写入数据读取

从本地数据缓存读取各 Skill 当日写入的运营数据：

| 数据分组 | 数据项 | 来源 Skill |
|---------|--------|-----------|
| 评价管理 | 新增差评数 / 已回复数 / 待处理数 / 新增好评数 / 好评自动回复数 / 硬件卫生差评数 | `hotel_review_manager` |
| 评价申诉 | 申诉发起数 / 通过数 / 驳回数 / 待结果数 | `hotel_review_appeal` |
| 促销调整 | 是否有促销调整 / 调整后当前折扣率 | `hotel_promotion_advisor` |
| 房态管理 | 自动下架次数 / 当前待上架房间数 | `hotel_room_switch` |
| 维修任务 | 新增维修任务数 / 已完成数 / 待处理数 / 长期待处理数 / 问题类型分布 / 来源分布 | `maintenance_reminder` |

> 当日某 Skill 无写入数据时，对应分组显示「昨日无记录」，不展示空数据行。

读取完成后跳转 §4 推送播报卡片。

## 4. 推送播报卡片

### 4.1 卡片输出格式

三行折叠结构，昨日数据默认展开，近7日 / 近30日折叠。
展开优先级根据商家前三次交互行为自动学习排列（见 §4.2）。

**昨日数据（默认展开）**：

```
┌──────────────────────────────────────┐
│ 📊 昨日经营播报 · 5月25日            │
│                                      │
│ ── 核心数据 ──                       │
│ 入住率          87%     ↑ vs 前日    │
│ 营收           ¥1,240   ↑ vs 前日    │
│ 间夜数          7间夜   → vs 前日    │
│ 平均每间夜收入  ¥177    ↑ vs 前日    │
│                                      │
│ ── 评价 ──                           │
│ 好评            5条     ↑ vs 前日    │
│ 差评            1条     ↓ vs 前日    │
│ 差评待处理      1条     ⚠️           │
│                                      │
│ ── 房态 ──                           │
│ 自动下架        1次                  │
│ 待上架房间      0间                  │
│                                      │
│ ── 维修 ──                           │
│ 新增任务        2条                  │
│ 待处理          1条     ⚠️           │
│                                      │
│ ── 促销 ──                           │
│ 昨日有调整      限时特价 88折 → 85折 │
│                                      │
│ ▶ 近7日汇总                          │
│ ▶ 近30日汇总                         │
└──────────────────────────────────────┘
```

> 趋势箭头说明：↑ 优于前日 / ↓ 差于前日 / → 持平（波动 < 1%）
> ⚠️ 标红提示项：差评待处理 / 维修待处理 / 待上架房间数 > 0

### 4.2 折叠展开优先级学习

**前三次播报**：近7日 / 近30日均折叠，记录商家每次点击展开的顺序：

```
第1次展开的行 → 优先级1
第2次展开的行 → 优先级2
```

**第四次起**：按学习到的优先级自动展开优先级1对应的行，其余保持折叠。

> 商家可在「播报设置」中手动重置学习结果。

## 6. 边界处理速查

| 场景 | 处理方式 |
|------|---------|
| 昨日无入住记录（空置日） | 推送「昨日暂无入住数据」，核心数据项显示 0，不展示空卡片 |
| 平台数据接口读取失败 | 推送「核心数据获取失败，请稍后重试」，各 Skill 写入数据正常展示 |
| 某 Skill 当日无写入数据 | 对应分组显示「昨日无记录」，不影响其他分组展示 |
| 有 ⚠️ 待处理项 | 卡片底部追加「有 X 项需要处理」提示，附对应 Skill 快捷入口 |
| 商家手动修改推送时间 | 覆盖默认设置，以手动设置为准 |
| 可售间夜数基准未配置 | 入住率 / 平均每间夜收入显示「—」，提示「请完善配置」 |
