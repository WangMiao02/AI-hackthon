# hotel_promotion_advisor — 促销方案调整

## 1. 执行入口判断

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 后台每日检测 | `trigger_type == "scheduled_check"`（每日固定时间触发） | → §2 竞品数据检测 |
| 商家主动唤起 | 商家在对话中提及特价 / 优惠 / 活动 / 涨价 / 调价等意图 | → §3 前置检查 |
| 配置缺失 | 首次使用或配置不完整 | → §6 Onboarding |

---

## 2. 竞品数据检测（每日后台静默执行）

### 2.1 拉取竞品数据

从美团民宿竞品洞察 API 读取：

| 数据项 | 口径 |
|--------|------|
| 周边竞品综合折扣率均值 | 3km 内同类型房源当前所有促销工具折扣率换算后的均值 |
| 商家当前综合折扣率 | 商家当前所有促销工具（限时特价/早鸟价/连住优惠）折扣率换算后的均值 |
| 竞品样本数量 | 参与计算的周边房源数，样本不足时标注 |

**折扣率统一换算口径**：
- 限时特价：直接取折扣率（如85折 = 85%）
- 早鸟价：取折扣率部分（提前天数不纳入折扣率计算）
- 连住优惠：按"减免金额 ÷ 原价总额"换算为等效折扣率

### 2.2 同期口径判断

```
当日属于节假日（含前后 2 日）→ 取周边同类节假日历史折扣率均值作为基准
当日属于非节假日            → 取周边近 7 日折扣率均值作为基准
```

> 节假日判断基于国家法定节假日日历，含元旦、春节、清明、五一、端午、中秋、国庆。

### 2.3 触发判断

```
触发条件：
周边综合折扣率均值 - 商家当前综合折扣率 ≥ 10 个百分点
```

**示例**：周边均值 82折，商家当前 92折 → 差距 10%，触发推送

**未触发**：本日不推送，等待次日检测

**触发**：进入 §4 生成推荐方案

## 4. 生成推荐方案

### 4.1 方案生成逻辑

基于以下维度综合生成建议调整方案：

| 维度 | 说明 |
|------|------|
| 竞品折扣率差距 | 差距越大，建议调整幅度越大 |
| 当前各促销工具状态 | 已开启 / 未开启 / 参数设置 |
| 近期入住率 | 入住率高时建议保守调整，入住率低时建议激进调整 |
| 商家利润空间 | 基于成本口径（见 §4.2）反推可承受的折扣幅度 |
| 同期口径 | 节假日 / 非节假日策略有所不同 |

**调整策略方向**：
- 未开启限时特价 → 优先建议开启，补齐与竞品最大的差距来源
- 已开启但力度不足 → 建议提高折扣率向竞品均值靠拢
- 早鸟价缺失 → 建议新增，适合节假日前提前锁客
- 连住优惠缺失 → 建议新增，适合淡季提升间夜数

### 4.2 利润计算口径

单间单夜利润 = 房间售价 × (1 - 折扣率) - 平台佣金 - 商家承担优惠金额

> 优惠承担说明：
> - 限时特价：平台与商家按约定比例各承担一部分，计算时仅扣除商家承担部分
> - 早鸟价 / 连住优惠：通常商家全额承担，从当前活动配置中读取实际比例

### 4.3 推送方案对比卡片

```

│ 周边均值 82折  你当前 92折  差距 10% │

│                                      │
│ ── 限时特价调整建议 ──               │
│                                      │
│          当前方案      建议方案      │
│ 折扣率    92折         88折          │
│ 商家承担  ¥24/间夜     ¥36/间夜     │
│                                      │
│ ── 连住优惠（当前未开启）──          │
│                                      │
│          当前方案      建议方案      │
│ 连住优惠  未开启       住2减30       │
│ 商家承担  —            ¥30/单        │
│                                      │
│ ── 利润预估 ──                       │
│                                      │
│ 单间夜利润  ¥156  →  ¥138           │
│ 预计入住率  ↑ 约12%（基于竞品数据） │
```

**按钮响应**：
- `全部采纳` → 调用 API 同时提交所有调整项，执行 §5

- `暂不调整` → 关闭卡片，记录本次推送已忽略，本周期不再重复推送

## 7. 边界处理速查

| 场景 | 处理方式 |
|------|---------|
| 当前无任何促销活动 | 推送「当前未设置任何促销，周边竞品均有活动」提示卡片，附建议新建方案 |
| 竞品洞察 API 读取失败 | 跳过本日检测，次日重新执行；连续 3 日失败则推送异常提示 |
| 周边竞品样本数不足（< 3 家） | 标注「周边参考房源不足，建议仅供参考」，不阻止推送 |
| API 提交调整失败 | 自动重试 1 次；仍失败则推送失败卡片，附 [手动前往榛果商家版] 按钮 |
| 商家忽略推送超过 3 次 | 降低检测频率为每 3 日一次，避免持续打扰 |
| 利润预估为负（折扣力度过大） | 卡片中标红警示「⚠️ 该方案单间夜利润为负，请谨慎采纳」，不阻止执行 |
| 节假日前 3 日内触发 | 卡片标注「⚠️ 节假日临近，调价后可能影响已询价客人预期」，不阻止执行 |
