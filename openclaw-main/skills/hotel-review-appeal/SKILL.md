# hotel_review_appeal — 房客评价申诉

## 1. 执行入口判断

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 上游 Skill 跳转 | 由 `hotel_review_manager` 差评卡片传入评价内容 | → §3 AI 问题分析（跳过评价读取） |
| 商家主动唤起 | 商家在对话中提及申诉 / 投诉 / 违规评价 / 客服介入等意图 | → §2 前置检查 |
| 商家主动跟进进度 | 商家询问"申诉结果怎么样了" | → §4 进度查询 |

---

## 2. 前置检查（商家主动唤起时执行）

1. 读取商家配置信息，**配置缺失** → 跳转 §6 Onboarding，Skill 结束
2. 拉取近 30 日内**未申诉的差评列表**（1-3星），推送选择卡片：

```
┌──────────────────────────────────────┐
│ 📋 选择要申诉的评价                  │
│                                      │
│ ① ⭐☆☆☆☆  5月23日  退房后2小时     │
│    「根本没住过，凭什么给我差评……」  │
│                                      │
│ ② ⭐⭐☆☆☆  5月21日  退房后1天       │
│    「什么破地方，骗子，投诉你们……」  │
│                                      │
│ ③ ⭐☆☆☆☆  5月19日  退房后3天       │
│    「照片和实际差太多，完全不一样……」│
│                                      │
│           [近30日已无差评]           │
└──────────────────────────────────────┘
```

> 商家点击某条评价后，传入评价内容，进入 §3 AI 问题分析。

---

## 3. AI 问题分析

### 3.1 分析评价内容

AI 读取评价文本，从以下类型中判断最匹配的问题类型：

| 问题类型 | 典型特征 |
|----------|---------|
| 未入住但留评 | 评价时间异常（取消订单后 / 退房时间极短）、内容与实际入住体验无关 |
| 疑似恶意差评 | 语气异常、内容模板化、无具体入住描述、疑似同行 |
| 内容含侮辱谩骂 | 包含人身攻击、侮辱性词汇 |
| 内容严重失实 | 描述与实际房源信息、订单记录明显不符 |
| 涉嫌敲诈勒索 | 以差评为要挟索要退款或额外补偿 |

同时生成一段**客服介入申请说明**供商家确认使用。

### 3.2 推送申诉确认卡片

```
┌──────────────────────────────────────┐
│ 🔍 AI 分析结果                       │
│                                      │
│ 原始评价                             │
│ 「根本没住过，凭什么给我差评，        │
│   要投诉你们……」                      │
│                                      │
│ 建议申诉类型                         │
│ ◉ 未入住但留评                       │
│ ○ 疑似恶意差评                       │
│ ○ 内容含侮辱谩骂                     │
│                                      │
│ 申请说明（可修改）                   │
│ ┌────────────────────────────────┐  │
│ │ 该评价用户订单显示已取消/未实  │  │
│ │ 际入住，评价内容与本房源无关，  │  │
│ │ 且含有不当言论，请平台核实处理 │  │
│ └────────────────────────────────┘  │
│                                      │
│    [准备好了，去联系客服]  [放弃]    │
└──────────────────────────────────────┘
```

**按钮响应**：
- `准备好了，去联系客服` → 执行 §4 引导提交
- `放弃` → 关闭卡片，Skill 结束

> 商家可在卡片内直接切换申诉类型、修改申请说明文本，无需重新发起。

---

## 4. 引导提交

平台暂无独立申诉 API，引导商家通过人工客服渠道提交：

1. 将申请说明复制到剪贴板
2. 推送引导操作卡片：

```
┌──────────────────────────────────────┐
│ 📋 申请说明已复制到剪贴板            │
│                                      │
│ 接下来请按以下步骤操作：             │
│                                      │
│ 1️⃣  打开榛果民宿商家版 App           │
│ 2️⃣  进入「我的」→「联系客服」        │
│ 3️⃣  说明「评价申诉」，粘贴说明内容  │
│ 4️⃣  等待客服处理（通常1-3个工作日）  │
│                                      │
│ 处理完成后记得回来告诉我结果         │
│                                      │
│  [打开榛果商家版]     [我知道了]     │
└──────────────────────────────────────┘
```

3. 记录申诉发起时间、评价 ID、申诉类型，写入待跟踪队列

---

## 5. 进度跟踪与结果处理

### 5.1 主动跟进提醒

申诉发起后 **3 个工作日**内未收到商家反馈结果，推送跟进提醒：

```
┌──────────────────────────────────────┐
│ ⏰ 申诉进度提醒                      │
│                                      │
│ 5月23日发起的申诉已超过3个工作日     │
│ 有结果了吗？                         │
│                                      │
│  [通过了 🎉]  [被驳回了]  [还在等]   │
└──────────────────────────────────────┘
```

### 5.2 商家反馈结果

**申诉通过**：

```
┌──────────────────────────────────────┐
│ 🎉 好消息！                          │
│                                      │
│ 申诉类型   未入住但留评              │
│ 处理结果   评价已由平台删除          │
│ 记录时间   05月26日 10:15            │
│                                      │
│           [知道了]                   │
└──────────────────────────────────────┘
```

**申诉驳回**：

```
┌──────────────────────────────────────┐
│ ❌ 申诉未通过                        │
│                                      │
│ 申诉类型   未入住但留评              │
│ 平台反馈   暂未发现违规，维持原评价  │
│ 记录时间   05月26日 10:15            │
│                                      │
│  [换个角度再试试]     [算了]         │
└──────────────────────────────────────┘
```

**按钮响应**：
- `换个角度再试试` → 返回 §3 申诉确认卡片，商家可修改申诉类型或说明后重新发起
- `算了` → 关闭卡片，Skill 结束

### 5.3 写入日报

无论通过或驳回，均将以下数据写入 `hotel_daily_report`：

| 数据项 | 说明 |
|--------|------|
| 当日申诉发起数 | 当日新发起的申诉总数 |
| 申诉通过数 | 当日商家确认通过的申诉数 |
| 申诉驳回数 | 当日商家确认驳回的申诉数 |
| 待结果申诉数 | 已发起尚未收到结果的申诉数 |

> 写入格式与口径以 `hotel_daily_report` §3 数据项定义为准。

## 7. 边界处理速查

| 场景 | 处理方式 |
|------|---------|
| 近30日无可申诉差评 | 推送「近30日暂无可申诉的差评」提示卡片 |
| 同一评价重复发起申诉 | 提示「该评价已有进行中的申诉，等有结果再说」，不允许重复发起 |
| 申诉说明内容为空 | 不允许进入引导步骤，提示「申请说明不能为空」 |
| 驳回后再次申诉内容未修改 | 提示「建议换个申诉类型或补充说明，提高通过率」，不强制阻止 |
| 商家长期未反馈结果（超过7个工作日） | 推送「申诉进度提醒」卡片，建议商家再次联系平台客服确认 |
