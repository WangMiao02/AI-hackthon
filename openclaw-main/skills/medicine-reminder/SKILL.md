# medicine_reminder — 药品补买提醒

## 1. 执行入口判断

**首先判断触发来源，然后跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 月度后台检查 | `trigger_type == "scheduled_check"`（每月1号 09:00） | → §2 库存计算 |
| 定时推送任务 | `trigger_type == "scheduled_push"`（动态日期触发） | → §3 推送提醒卡片 |
| 用户主动唤起 | 对话中含药品/补药/买药等意图 | → §2 前置检查 |
| 用户未完成配置 | 药品信息缺失 | → §4 Onboarding |

---

## 2. 前置检查（用户主动触发时执行）

1. 读取用户药品配置信息
2. 如果**配置缺失** → 跳转 §4 Onboarding，Skill 结束
3. 如果**配置完整** → 跳转 §3 推送提醒卡片

---

## 3. 月度后台检查（静默，不推送）

> 仅 `scheduled_check` 触发时执行，用户无感知。

**执行步骤**：

1. 读取用户预填药品清单（药名、每日用量、当前库存数量）
2. 读取美团买药历史订单，辅助修正库存估算（预填数据优先，历史订单作参考）
3. 对每种药品计算预测耗尽日：

   ```
   预测耗尽日 = 今日 + （当前库存数量 ÷ 每日用量）
   实际推送日 = 预测耗尽日 - 3天
   ```

4. 如果推送日 ≤ 今日（库存已严重不足）→ 立即触发 §3 推送
5. 否则 → 将推送任务写入调度队列，到期自动触发 §3

---

## 4. 推送提醒卡片

### 3.1 卡片输出格式

```
┌──────────────────────────────────────┐
│ 💊 有 2 种药即将用完，记得补货       │
│                                      │
│ 降压药（苯磺酸氨氯地平）             │
│ 预计 5月15日 用完 · 还剩 3 天        │
│                                      │
│ 维生素D                              │
│ 预计 5月18日 用完 · 还剩 6 天        │
│                                      │
│    [帮我下单]      [稍后提醒]        │
└──────────────────────────────────────┘
```

**用户响应分支**：
- 点击 `帮我下单` → 传入药品清单，触发 `skill_order`，Skill 结束
- 点击 `稍后提醒` → 推迟24小时再次推送，Skill 结束
- 无响应 → 不重复打扰，下次月度检查重新计算

---

## 5. Onboarding（首次配置）

引导用户逐一填写药品信息：

| 配置项 | 示例 |
|--------|------|
| 药品名称 | 苯磺酸氨氯地平 |
| 每日用量 | 1片 |
| 当前库存数量 | 12片 |
| 月度检查日（默认每月1号） | 每月1号 |

> 已配置用户可在「健康设置」中增删药品或修改月度检查日。
> 补买完成后需手动更新库存数量，或通过美团买药订单自动同步（需授权）。

---

## 6. 边界处理速查

| 场景 | 处理 |
|------|------|
| 用户未填写药品信息 | 触发 Onboarding 引导，Skill 暂不执行 |
| 库存充足（本月内不会耗尽） | 本月不推送，下次检查日重新计算 |
| 推送日已过（库存严重不足） | 跳过调度队列，立即推送 |
| 用户点击「稍后提醒」 | 推迟24小时重推，最多推迟2次 |

---

## 7. 本期不做（Backlog）

> 以下功能**不在当前 Skill 范围内**，遇到请告知用户暂不支持：

- 处方药核验 / 用药建议（医疗功能）
- 多人用药管理（家庭成员各自用药）
- 用药提醒（每日按时服药提醒）
- 药品价格比价
