# 结果计算与联动 — 卡片模板与交互细节

## 时间热力图（辅助展示）

在结果卡片之前，可选展示一个简化的时间热力图，让发起人直观看到重叠情况：

```
┌──────────────────────────────────────┐
│ 📊 时间热力图                        │
│                                      │
│          5/31 周六    6/1 周日       │
│ 17:00    ██░░        ░░░░           │
│ 17:30    ██░░        █░░░           │
│ 18:00    ████        ██░░           │
│ 18:30    ████        ██░░           │
│ 19:00    ████        ███░           │
│ 19:30    ███░        ███░           │
│ 20:00    ███░        ████           │
│ 20:30    ██░░        ████           │
│ 21:00    █░░░        ███░           │
│ 21:30    ░░░░        ██░░           │
│                                      │
│ ████ = 4/4人可用  ███░ = 3/4人可用  │
│ ██░░ = 2/4人可用  █░░░ = 1/4人可用  │
│ ░░░░ = 无人可用                      │
└──────────────────────────────────────┘
```

## 情况 A：存在全员可用时段

```
┌──────────────────────────────────────┐
│ 🎉 找到大家都有空的时间！            │
│                                      │
│ 📅 周末聚餐（4人参与）              │
│                                      │
│ ✅ 最佳时段（最长连续可用）：        │
│    周六 18:00-20:00（4/4 人可用）    │
│                                      │
│ 🟡 其他全员可用时段：                │
│    周六 19:00-20:30（4/4 人可用）    │
│    周日 20:00-22:00（4/4 人可用）    │
│                                      │
│ [就定周六 18:00]  [选其他时段]       │
└──────────────────────────────────────┘
```

**最佳时段排序规则：**

| 优先级 | 排序维度 | 说明 |
|--------|---------|------|
| 1 | 可用人数 | 全员可用 > 缺 1 人 > 缺 2 人 |
| 2 | 连续时长 | 连续可用时间越长越好（聚餐至少需要 1.5-2 小时） |
| 3 | 时间靠前 | 同等条件下优先推荐更早的时段 |

**用户响应分支：**

| 操作 | 处理 |
|------|------|
| 点击「就定周六 18:00」 | 确认时间 → §7 联动下游 Skill |
| 点击「选其他时段」 | 展开所有全员可用时段供选择 |

```
│                                      │
│ 🥈 周日 19:00-21:00（3/4 人可用）   │
│    ✅ Sarah  ✅ 小王  ✅ 小张        │
│    ❌ 小李                           │
│                                      │
│ 🥉 周六 20:00-22:00（3/4 人可用）   │
│    ✅ Sarah  ✅ 小李  ✅ 小张        │
│    ❌ 小王                           │
│                                      │
│ 你想怎么办？                         │
│                                      │
│ [就定方案一，3人先约]                │
│ [问问小张周六能不能调]               │
│ [扩大日期范围重新收集]               │
│ [算了，下次再约]                     │
└──────────────────────────────────────┘
```

**用户响应分支：**

| 操作 | 处理 |
|------|------|
| 「就定方案一，3人先约」 | 确认时间（3人版） → §7 联动下游 Skill |
| 「问问小张周六能不能调」 | 生成询问文案供用户复制（见下） |
| 「扩大日期范围重新收集」 | 回到 §2，保留已有数据，扩展日期范围 |
| 「算了，下次再约」 | Skill 结束 |

**询问文案（供用户复制）**：

```
📋 已复制到剪贴板：

"小张～我们想约周六 18:00-20:00 聚餐，
 你那个时间实在调不开吗？
 如果能来的话告诉我一声~"
```

## 确认时间 → 联动下游 Skill

用户确认最终时间后的联动卡片：

### 约局目的已知时（如"聚餐"）

```
┌──────────────────────────────────────┐
│ ✅ 时间定了！                        │
│                                      │
│ 📅 周六 18:00-20:00                  │
│ 👥 4 人：Sarah、小王、小李、小张     │
│ 🎯 聚餐                              │
│                                      │
│ 接下来帮你搞定去哪吃？              │
│                                      │
│ [帮我找餐厅订座]                     │
│  → 按 4 人、周六 18:00 筛选餐厅     │
│                                      │
│ [发投票让大家选餐厅]                 │
│  → 我先推荐几家，大家投票决定        │
│                                      │
│ [点外卖拼单]                         │
│  → 发起拼单，大家各选各的            │
│                                      │
│ [不用了，时间定了就行]               │
└──────────────────────────────────────┘
```

### 约局目的未知时

```
┌──────────────────────────────────────┐
│ ✅ 时间定了！                        │
│                                      │
│ 📅 周六 18:00-20:00                  │
│ 👥 4 人：Sarah、小王、小李、小张     │
│                                      │
│ 时间搞定了～你们打算干嘛？           │
│                                      │
│ [去吃饭]  → 找餐厅 / 投票选         │
│ [看电影]  → 查周六晚上的场次         │
│ [看演出]  → 查周六晚上的演出         │
│ [逛景区]  → 查周六可去的地方         │
│ [还没想好，发投票让大家选]           │
│                                      │
│ [不用了，时间定了就行]               │
└──────────────────────────────────────┘
```

**用户响应分支：**

| 操作 | 联动 Skill | 预填参数 |
|------|-----------|---------|
| 帮我找餐厅订座 | `restaurant_reservation` | 时间=周六18:00, 人数=4 |
| 发投票让大家选餐厅 | `preference_poll` | 类型=餐厅, 人数=4 |
| 点外卖拼单 | `group_order` | 人数=4 |
| 看电影 | `movie_ticket` | 日期=周六, 时段=18:00后, 人数=4 |
| 看演出 | `concert_reserver` | 日期=周六 |
| 逛景区 | `attraction_ticket` | 日期=周六 |
| 还没想好，发投票 | `preference_poll` | 意图收敛模式（§0.3） |
| 不用了 | — | Skill 结束 |

## 通知参与者最终时间（可选）

用户确认时间后，可选择通知所有参与者：

```
┌──────────────────────────────────────┐
│ 📢 要通知大家最终时间吗？            │
│                                      │
│ 📋 通知文案预览：                    │
│                                      │
│ "时间定了！周六 18:00 聚餐，         │
│  到时候见～具体去哪吃我再通知大家"   │
│                                      │
│ [复制文案发给大家]  [不用通知]       │
└──────────────────────────────────────┘
```

> 不自动发送通知，仅生成文案供用户复制。尊重用户的社交主动权。

## 部分人确认时的特殊处理

当用户选择"3 人先约"（有人缺席）时：

```
┌──────────────────────────────────────┐
│ ✅ 时间定了！（3人版）               │
│                                      │
│ 📅 周六 18:00-20:00                  │
│ 👥 3 人：Sarah、小王、小李           │
│ ⚠️ 小张这次来不了                    │
│                                      │
│ 接下来帮你搞定去哪吃？              │
│                                      │
│ [帮我找餐厅订座]                     │
│ [发投票让大家选餐厅]                 │
│ [不用了，时间定了就行]               │
│                                      │
│ 💡 要不要给小张发个消息说一声？      │
│    [生成通知文案]  [不用]            │
└──────────────────────────────────────┘
```

通知文案示例：
```
"小张～这次我们定了周六 18:00 聚餐，
 你那天不方便就下次一起呀！"
```
