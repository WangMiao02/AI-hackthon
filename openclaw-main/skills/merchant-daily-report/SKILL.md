# merchant_daily_report — 每日营业数据播报

## 1. 执行入口判断

**首先判断触发来源，然后跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时系统触发 | `trigger_type == "scheduled"`（时间见§2） | → §3 数据拉取 |
| 商家主动唤起 | 对话中含营业额/订单/数据等意图 | → §2 前置检查 |
| 商家未完成配置 | Onboarding 信息缺失 | → §5 Onboarding |

---

## 2. 前置检查（商家主动触发时执行）

1. 读取商家 Onboarding 配置（商家类型、推送时间）
2. 如果**配置缺失** → 跳转 §5 Onboarding，Skill 结束
3. 如果**配置完整** → 跳转 §3 数据拉取

**定时推送时间规则**：

| 商家类型 | 默认推送时间 | 判断方式 |
|---------|------------|---------|
| 早餐商家 | 每日 05:00 | 系统根据历史营业时间自动识别 |
| 普通商家（10点后开业） | 每日 08:00 | 系统自动识别 |
| 24小时商家 | 每日 08:00 | 系统自动识别 |

> 商家可在「播报设置」中手动修改推送时间，覆盖系统自动识别结果。

---

## 3. 数据拉取（静默）

从美团商家后台接口读取以下数据：

| 数据项 | 口径说明 |
|--------|---------|
| 到手营业额 | 扣除平台佣金 + 骑手佣金后的实收金额 |
| 订单量 | 完成订单总数（不含退单） |
| 退单率 | 退单数 ÷ 总下单数 |
| 好评数 | 当日新增好评条数 |
| 客单价 | 到手营业额 ÷ 完成订单量 |

读取三个周期：昨日、近7日、近30日，跳转 §4 推送播报卡片

---

## 4. 推送播报卡片

### 4.1 卡片输出格式

三行折叠结构，默认展开顺序由 §4.2 展开优先级决定。

**昨日数据（示例）**：
```
┌──────────────────────────────────────┐
│ 📊 昨日营业播报 · 5月20日            │
│                                      │
│ 到手营业额   ¥3,280   ↑ vs 前日      │
│ 订单量       128单    ↓ vs 前日      │
│ 退单率       2.3%     → vs 前日      │
│ 好评数       47条     ↑ vs 前日      │
│ 客单价       ¥25.6    ↑ vs 前日      │
│                                      │
│ ▶ 近7日汇总                          │
│ ▶ 近30日汇总                         │
└──────────────────────────────────────┘
```

> 趋势箭头说明：↑ 优于前日 / ↓ 差于前日 / → 持平（波动<1%）

**点击「近7日汇总」展开**：
```
├ 近7日汇总
│ 到手营业额   ¥21,450   ↑ vs 上周同期
│ 订单量       836单     ↑ vs 上周同期
│ 退单率       2.1%      ↓ vs 上周同期
│ 好评数       312条     ↑ vs 上周同期
│ 客单价       ¥25.7     → vs 上周同期
```

**点击「近30日汇总」展开**：
```
├ 近30日汇总
│ 到手营业额   ¥89,200   ↑ vs 上月同期
│ 订单量       3,421单   ↑ vs 上月同期
│ 退单率       2.4%      ↓ vs 上月同期
│ 好评数       1,205条   ↑ vs 上月同期
│ 客单价       ¥26.1     ↑ vs 上月同期
```

### 4.2 折叠展开优先级学习

**前三次播报**：三行均折叠，记录商家每次点击展开的顺序：

```
第1次展开的行 → 优先级1
第2次展开的行 → 优先级2
第3次展开的行 → 优先级3
```

**第四次起**：按学习到的优先级自动展开优先级1对应的行，其余保持折叠。

> 学习结果由 `profile_updater`（B端）写入 `agent_inferred` 字段，商家可在「播报设置」中手动重置。

---

## 5. Onboarding（首次配置）

引导商家完成以下配置：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 商家类型 | 系统自动识别，商家可修改 | 普通商家 / 早餐商家 / 24小时 |
| 推送时间 | 默认按商家类型，可手动覆盖 | 08:00 |

> 配置完成后自动跳转 §3 执行首次播报。已配置商家可在「播报设置」中修改。

---

## 6. 边界处理速查

| 场景 | 处理 |
|------|------|
| 商家未完成 Onboarding | 触发 §5 引导，Skill 暂不执行 |
| 昨日无订单（休息日/临时关店） | 推送「昨日暂无营业数据」，不展示空数据卡片 |
| 数据接口读取失败 | 提示「数据获取失败，请稍后重试」 |
| 前三次未完成展开行为 | 保持默认折叠，继续累积学习，不强制推断 |
| 商家手动修改推送时间 | 覆盖系统自动识别结果，以手动设置为准 |

---

## 7. 本期不做（Backlog）

> 以下功能**不在当前 Skill 范围内**，遇到请告知商家暂不支持：

- 食材成本、包装费等内部成本录入与利润核算
- 菜品维度数据（单品销量、爆款分析）
- 竞品对比数据
- 数据异常智能预警（如某日退单率突然飙升）
- 自定义播报数据项
