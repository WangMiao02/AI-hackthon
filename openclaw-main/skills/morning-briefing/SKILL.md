# morning_briefing — 早间资讯推送

## 1. 执行入口判断

**首先判断触发来源，跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时系统触发 | `trigger_type == "scheduled"`（每日 7:30） | → §2 数据聚合 |
| 用户主动唤起 | 对话中含早报/天气/新闻/资讯等意图 | → §2 数据聚合 |
| 用户未完成配置 | 首次使用，常驻城市未设置 | → §6 Onboarding |

---

## 2. 数据聚合

**并行拉取以下数据，超时（3s）的模块降级处理：**

| 模块 | 数据来源 | 降级方案 |
|------|---------|---------|
| 天气 | 天气接口（用户城市） | 展示「天气数据获取失败，请查看天气 App」 |
| 今日行程 | 用户当日行程接口（汇总所有 skill 写入的预约） | 跳过该模块，不展示 |
| 当地新闻 | 当地新闻聚合接口 | 跳过该模块，不展示 |
| 全国大事 | 热点新闻接口 | 跳过该模块，不展示 |
| 美团福利活动 | 美团内部福利接口 | 跳过该模块，不展示 |
| 个性化内容 | 用户兴趣标签接口 | 降级展示平台热门内容 |

数据聚合完成后 → §3 内容组装与推送

---

## 3. 内容组装与推送

**组装规则**（按优先级排列）：

| 优先级 | 模块 | 展示条数 | 说明 |
|--------|------|---------|------|
| 1 | 天气 | 固定展示 | 当日天气 + 未来 3 天趋势 |
| 2 | 今日行程 | 有则全展示 | 当日所有预约（美甲/美发/按摩/餐厅等），每条附「帮我叫车」按钮 |
| 3 | 美团福利活动 | 最多 2 条 | 员工福利/平台活动，有则必展示 |
| 4 | 全国大事 | 最多 2 条 | 今日重要热点 |
| 5 | 当地新闻 | 最多 2 条 | 用户所在城市本地新闻 |
| 6 | 个性化内容 | 最多 2 条 | 基于用户兴趣标签推荐 |

> 总条数控制在 **5–8 条**，避免信息过载。今日行程和美团福利活动若无内容则跳过，不占位。

推送卡片格式详见 [references/briefing-card.md](references/briefing-card.md)

---

## 4. 用户响应处理

推送卡片后，等待用户交互：

| 用户操作 | 处理 |
|---------|------|
| 点击「帮我订早餐」 | 触发 `food_recommend` skill，场景标注为「早餐」 |
| 点击行程条目「帮我叫车」 | 询问「你打算几点出发？」，用户填写出发时间后触发 `ride_hailing`，传入目的地 + 出发时间 |
| 点击「帮我叫车」（无行程） | 直接触发 `ride_hailing` skill |
| 点击某条新闻「查看详情」 | 展示该条新闻摘要（3–5 句话），底部附原文链接 |
| 点击「展开更多」 | 展开折叠的资讯条目 |
| 无操作 | 静默，不重复推送 |

> **恶劣天气主动联动**：天气模块检测到雨天/高温/大风/雾霾/雪天时，在天气区块底部自动追加「今天天气不好，需要帮你叫车吗？[帮我叫车] [不用了]」，用户当日选择「不用了」后不再重复询问。

> **今日行程叫车逻辑**：行程模块中每条预约旁附「帮我叫车」按钮，点击后**先询问出发时间**（不预设默认值），用户确认后再触发 `ride_hailing`。避免在预约当天过早推送叫车请求。

---

## 5. 推送时间管理

| 场景 | 处理 |
|------|------|
| 默认推送时间 | 每日 7:30 |
| 用户自定义时间 | 在 Onboarding 或设置中修改，范围 6:00–9:00 |
| 用户当日已主动唤起 | 定时推送静默跳过，避免重复 |
| 节假日 | 正常推送，内容中标注节日信息 |

---

## 6. Onboarding（首次配置）

| 配置项 | 示例 | 说明 |
|--------|------|------|
| 常驻城市 | 北京 | 天气和当地新闻的默认城市 |
| 感兴趣的内容领域 | 科技 / 财经 / 体育 / 娱乐 / 社会（多选） | 个性化内容推荐依据 |
| 推送时间 | 07:30 | 每日定时推送时间，范围 6:00–9:00 |
| 是否接收美团福利活动 | 是 / 否 | 默认开启 |

> 配置完成后立即推送一次当日早报。已配置用户可在「早报设置」中修改。

---

## 7. 边界处理速查

| 场景 | 处理 |
|------|------|
| GPS 未授权 | 降级使用 Onboarding 中的常驻城市 |
| 常驻城市未设置 | 触发 §6 Onboarding |
| 天气接口超时 | 展示「天气数据获取失败」，其余模块正常展示 |
| 新闻接口全部超时 | 仅展示天气 + 美团福利活动 + 联动按钮 |
| 美团福利活动无内容 | 跳过该模块，不占位 |
| 用户兴趣标签为空 | 降级展示平台热门内容 |
| 用户当日已主动唤起过 | 定时推送静默跳过 |
| 用户关闭早报推送 | 停止定时触发，用户主动唤起仍可使用 |
