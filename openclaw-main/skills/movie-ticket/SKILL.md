# movie_ticket — 电影票预订

## 1. 执行入口判断

**首先判断触发来源，跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 用户主动唤起 | 对话中含电影/影院/电影票等意图 | → §2 意图识别与分流 |
| 用户未完成配置 | 首次使用，常用影院未设置 | → §6 Onboarding |

---

## 2. 意图识别与分流

| 用户意图 | 示例表达 | 跳转 |
|---------|---------|------|
| 发现/推荐电影 | "最近有什么好看的""今天有什么电影" | → §3 电影查询（推荐模式） |
| 查询特定电影 | "XX电影在哪里放""XX电影几点有场" | → §3 电影查询（带关键词） |
| 直接购票 | "帮我买两张XX电影票""订今晚的票" | → §4 场次与选座 |

---

## 3. 电影查询与推荐

**输入**：用户城市（GPS/常用城市）+ 关键词/偏好 + 观影日期

**执行步骤**：

1. 确定查询范围：有关键词 → 精确搜索；无关键词 → 基于偏好推荐
2. 调用猫眼电影接口，按以下规则排序：

   | 优先级 | 筛选维度 | 说明 |
   |--------|---------|------|
   | 1 | 用户偏好匹配 | 历史观影类型命中（动作/爱情/科幻等） |
   | 2 | 上映状态 | 正在热映优先，即将上映次之 |
   | 3 | 评分/口碑 | 猫眼评分 + 想看人数 |
   | 4 | 价格匹配 | 落在用户预算区间内 |

3. 取前 3 条展示，超出折叠；无结果 → 提示调整条件
4. 卡片格式及交互详见 [references/search-flow.md](references/search-flow.md)

---

## 4. 场次查询与选座

**执行步骤**：

1. 确认电影 + 观影日期（默认今天）
2. 查询附近影院的可用场次，按距离 + 价格排序
3. 用户选择场次后，展示座位图（行列选择）
4. 确认座位 + 购票数量 → 推送确认卡片 → 调用 `order` skill 完成支付

**冷启动处理**（无历史偏好）：默认展示距离最近的 3 家影院

卡片格式及交互详见 [references/purchase-flow.md](references/purchase-flow.md)

---

## 5. 购票结果处理

**成功**：
```
✅ 购票成功！
🎬 [电影名称] · [影院名称]
📅 [日期] [场次时间] · [厅型]
💺 [座位信息] × [数量]张
💰 ¥[金额]
🎟 取票码：[查看二维码]
📌 观影前 30 分钟我会提醒你。
```
→ 设置观影前 30 分钟提醒任务
→ 如影院距当前位置 > 2km → 推送联动打车建议：`[叫车去影院] [不用了]`
→ 点击「叫车去影院」→ 触发 `ride_hailing`，目的地预填为影院地址

**失败**：
→ 提示失败原因，保留座位信息，引导重试或重新选座

## 6. Onboarding（首次配置）

| 配置项 | 示例 | 说明 |
|--------|------|------|
| 常驻城市 | 北京 | 默认搜索范围 |
| 偏好影院 | 万达影城 / 无偏好 | 场次查询时优先展示 |
| 感兴趣的电影类型 | 科幻、动作、爱情 | 推荐筛选依据 |
| 座位偏好 | 中间排 / 后排 / 无偏好 | 选座时自动高亮推荐区域 |
| 预算区间 | ¥30–¥80 | 票价过滤 |

> 配置完成后自动跳转 §3 推荐流程。

---

## 7. 边界处理速查

| 场景 | 处理 |
|------|------|
| 目标场次已售罄 | 提示「该场次已售罄」，推荐相邻场次 |
| 选座时座位被抢占 | 提示「该座位刚被选走」，引导重新选座 |
| 支付超时（10分钟） | 推送催促，超时后订单自动取消，座位释放 |
| GPS 未授权 | 降级使用 Onboarding 中的常驻城市 |
| 附近无影院 | 扩大搜索范围至 10km，提示距离 |
| 电影已下映 | 提示「该电影已下映」，推荐同类型在映电影 |
