# nail_salon — 美甲预约

## 1. 执行入口判断

**首先判断触发来源，跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时系统触发（周期复约） | `trigger_type == "scheduled"` + 周期提醒任务存在 | → §5 周期复约提醒 |
| 用户主动唤起 | 对话中含美甲/美睫/纹绣等意图 | → §2 意图识别与分流 |
| 用户未完成配置 | 首次使用，常用地址未设置 | → §6 Onboarding |

---

## 2. 意图识别与分流

| 用户意图 | 示例表达 | 服务类型 | 跳转 |
|---------|---------|---------|------|
| 美甲 | "做指甲""换个甲色""想做美甲" | `nail` | → §3 门店查询 |
| 美睫 | "接睫毛""美睫""嫁接睫毛" | `lash` | → §3 门店查询 |
| 纹绣 | "纹眉""飘眉""半永久眉毛" | `embroidery` | → §3 门店查询 |
| 意图不明确 | "帮我约个丽人项目" | — | 追问：「想做美甲、美睫还是纹绣？」 |

> **美甲款式偏好**：意图识别为 `nail` 后，可选追问「有偏好的风格或款式吗？」，接受用户的**任意自由文本输入**（如"海莉甲""猫眼甲""奶油甲""法式"等），将其作为关键词传入门店查询接口，模糊匹配有对应作品标签的门店。用户也可跳过直接看门店。若用户在触发时已说明款式（如"我想做海莉甲"），则直接提取款式关键词，无需再追问。

---

## 3. 门店查询与推荐

**输入**：服务类型 + 用户地址（GPS/常用地址）+ 款式风格偏好（如有）

**执行步骤**：

1. 调用美团丽人接口，查询附近门店及技师
2. 按以下规则排序，取前 **2 家** 展示，超出折叠：

   | 优先级 | 筛选维度 | 说明 |
   |--------|---------|------|
   | 1 | 历史消费过该门店 | 熟悉环境，用户信任度高 |
   | 2 | 款式关键词匹配 | 用关键词模糊匹配门店作品标签（支持任意款式名，如"海莉甲""猫眼甲"等） |
   | 3 | 评分 | 4.8分及以上优先 |
   | 4 | 距离 | 同等条件取距离最近 |

   **冷启动处理**（无历史消费记录）：按综合评分从高到低排列，展示评分 + 服务次数 + 人均价格

3. 无可用门店 → 提示「附近暂无匹配的美甲门店，请尝试调整位置」
4. 有结果 → 跳转 §4 预约确认

---

## 4. 预约确认

**执行步骤**：

1. 展示门店推荐卡片（含评分、人均价格、距离、**精选款式缩略图**）
2. 用户选择门店 → 展示该店款式作品图集（按风格分类）
3. 用户选择感兴趣的款式（或跳过）→ 展示可选技师（含评分及其作品图）
4. 用户选择技师（或跳过由系统分配）→ 展示可选项目及价格
5. 用户选择项目 → 展示可选时段
6. 用户选择时段 → 展示预约确认卡片
7. 用户确认 → 调用 `order` skill 完成支付 → §4.1 预约结果处理

> **款式作品图加载规则**：从门店/技师的评论区及作品集拉取最近作品图，按风格标签分组展示（法式/ins风/中式/简约/华丽），每组展示 3 张缩略图，点击可放大并查看评论原文。图片加载失败时降级展示文字风格标签。

卡片格式及款式图交互详见 [references/booking-flow.md](references/booking-flow.md)

### 4.1 预约结果处理

**成功**：
```
✅ 预约成功！
💅 [项目名称] · [门店名称]
👤 技师：[技师姓名] ⭐[评分]
📅 [预约日期] [时间段]
📍 [门店地址]
💰 ¥[金额]
📌 到店前 30 分钟会提醒你出发。
```
→ 将预约信息写入当日行程，供 `morning_briefing` 早报「今日行程」模块读取，统一处理叫车提醒
→ 预约成功卡片底部追加轻量收尾文案：「下次想做美甲，随时来找我预约～」，不强制询问周期；周期复约设置入口保留在 §5，用户主动说「帮我设置提醒」时才触发

**失败**：
→ 提示失败原因，保留预约信息，引导重试或换门店/时段

## 5. 周期复约提醒

### 5.1 首次设置（用户主动触发）

用户说「帮我设置美甲提醒」「下次提前提醒我」等时触发，询问：

```
🔄 好的！多久提醒你一次？

[3周后]  [1个月后]  [自定义]
```

用户选择周期后，记录以下信息用于下次复约：
- 服务类型（`nail` / `lash` / `embroidery`）
- 上次门店信息（优先复用）
- 上次技师信息（优先复用）
- 上次项目信息
- 下次提醒时间 = 当前日期 + 周期 - 1天

### 5.2 到期提醒推送（定时触发）

到期前一天推送复约卡片，格式及交互详见 [references/periodic-reminder.md](references/periodic-reminder.md)

> 配置完成后自动跳转 §3 门店查询。已配置用户可在「美甲设置」中修改。

## 7. 边界处理速查

| 场景 | 处理 |
|------|------|
| 附近无可用门店 | 提示调整位置或扩大搜索范围 |
| 用户选择时段已被预约 | 提示「该时段刚被预约」，引导重新选择 |
| 支付失败 | 提示原因，保留预约信息可重试 |
| 门店临时闭店 | 推送通知，引导重新选择门店 |
| 常用地址未设置 | 触发 §6 Onboarding |
| GPS 未授权 | 降级使用 Onboarding 中的常用地址 |
| 款式作品图加载失败 | 降级展示文字风格标签 |
| 纹绣类需确认是否首次 | 首次纹绣需提示「建议到店咨询后再预约，部分门店需先设计再预约」 |
| 周期提醒到期但用户已手动预约 | 检测到近期已有预约记录则静默跳过 |

## 8. 本期不做（Backlog）

- 上门美甲服务
- 美甲材料/工具购买
- 多人同时预约（闺蜜同行）
- 门店会员卡/储值卡管理
- 款式收藏夹（用户保存喜欢的款式）
- 效果评价/晒图
- 退款/改约流程
- 跨平台比价
