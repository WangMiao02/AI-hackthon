# order — 下单执行

## 1. 执行入口判断

**首先判断触发来源，然后跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 上游 Skill 联动触发 | `trigger_type == "linked"` + 商品清单已传入 | → §2 确认页展示 |
| 用户主动唤起 | 对话中含下单/购买/买单等意图 | → §2 确认页展示 |

> **联动触发时**：从上游 Skill 上下文中读取 `order_items`（商品清单）和 `order_type`（品类标识），直接进入 §2，无需用户重复输入。

---

## 2. 确认页展示

根据 `order_type` 渲染对应确认页格式：

### 2.1 外卖订单确认页

```
┌──────────────────────────────────────┐
│ 🍜 确认下单                          │
│                                      │
│ 黄焖鸡米饭（大份）                   │
│ 来自：老乡鸡（静安寺店）             │
│ 配送时长：约 30 分钟                 │
│ 餐品小计：¥28.00                     │
│ 配送费：¥5.00                        │
│ 合计：¥33.00                         │
│ 支付方式：美团支付（默认）           │
│                                      │
│      [确认下单]     [取消]           │
└──────────────────────────────────────┘
```

### 2.2 商品订单确认页（药品 / 日用品）

```
┌──────────────────────────────────────┐
│ 🛒 确认下单                          │
│                                      │
│ 苯磺酸氨氯地平片 × 2盒   ¥ 48.00    │
│ 维生素D滴剂 × 1盒        ¥ 32.00    │
│                                      │
│ 合计：¥80.00                         │
│ 预计送达：明日上午                   │
│ 支付方式：美团支付（默认）           │
│                                      │
│      [确认下单]     [取消]           │
└──────────────────────────────────────┘
```

**用户响应分支**：
- 点击 `确认下单` → 调用下单接口 + 支付 → §3 下单结果处理
- 点击 `取消` → 返回上游 Skill 卡片，Skill 结束

---

## 3. 下单结果处理

**成功**：
```
✅ 下单成功！

老乡鸡（静安寺店）· 黄焖鸡米饭（大份）
预计 30 分钟送达 · ¥33.00
```

**失败**：
→ 提示失败原因，保留当前订单信息，引导用户重试或返回上游重新选择

---

## 4. 边界处理速查

| 场景 | 处理 |
|------|------|
| 上游未传入商品清单 | 提示「下单信息缺失，请重新选择」，Skill 结束 |
| 支付失败 | 提示失败原因，保留订单信息可重试 |
| 商品下架 / 餐厅打烊 | 提示「该商品暂时无法购买」，引导返回上游重选 |
| 配送地址未设置 | 提示用户补充收货地址后重试 |

---

## 5. 本期不做（Backlog）

> 以下功能**不在当前 Skill 范围内**，遇到请告知用户暂不支持：

- 退款 / 退票 / 售后流程
- 多地址管理（下单时切换收货地址）
- 优惠券自动核销（下单时自动匹配最优券）
- 订单状态实时追踪
