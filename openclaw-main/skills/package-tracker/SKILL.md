# package_tracker — 快递追踪管家

## 0. Demo 模式声明

本 Skill 运行于 Demo 模式。物流数据来源于本地 mock 数据库（`references/mock-packages.json`），
模拟快递聚合 API 返回结果。数据结构与真实快递100/菜鸟接口对齐，演示完整的查询-提醒-操作闭环。

**接口模拟方式**：
```
GET /api/packages?user_id=demo_user
Response: references/mock-packages.json
```

---

## 1. 执行入口判断

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时触发（驿站超时预警） | `trigger_type == "station_timeout"` | -> S4 超时预警推送 |
| 定时触发（出差期间到件） | `trigger_type == "trip_arrival"` | -> S5 出差到件提醒 |
| 用户主动查询 | 对话中含快递/包裹/物流/取件等意图 | -> S2 前置数据拉取 |
| 用户确认签收 | 对话中含"已经取了"/"签收了" | -> S6 签收确认 |

---

## 2. 前置数据拉取

**执行步骤**：

1. 调用快递聚合接口：`GET /api/packages?user_id=demo_user`
2. 解析返回数据，按状态分组：
   - `delivering`（派送中）— 最紧急，优先展示
   - `station_arrived`（已到驿站）— 需要操作，次优先
   - `in_transit`（运输中）— 信息类，最后展示
   - `signed`（已签收）— 默认不展示，用户追问时可查
3. 检查是否有超时风险包裹（`station_deadline` 距当前时间 < 4小时）
4. 数据就绪 -> 跳转 S3 展示

## 4. 驿站超时预警推送

> 定时检查触发：每小时扫描一次，发现 `station_deadline` 距当前 < 2小时 的包裹时推送。

```
┌──────────────────────────────────────┐
│ ⚠️ 快递即将超时                       │
│                                      │
│ [商品名]                             │
│ [平台] · [快递公司]                   │
│ 存放：[驿站/柜名称]                   │
│ 超时时间：[具体时间]（还剩 [X] 小时） │
│ 超时后收费：1元/天                    │
│                                      │
│    [我去取]  [帮我延期]  [设置代收]   │
└──────────────────────────────────────┘
```

**用户响应分支**：

| 操作 | 处理 |
|------|------|
| 我去取 | 标记为"用户已知"，不再重复提醒 |
| 帮我延期 | 提示：大部分驿站不支持线上延期，建议尽快取件或联系驿站 |
| 设置代收 | 询问代收人信息，记录到 memory 供下次使用 |

---

## 5. 出差期间到件提醒

> 联动 trip_reserver：当用户有出差行程（不在常住城市）期间，有包裹状态变为 `station_arrived` 时触发。

**触发条件**：
1. memory 中存在 trip_reserver 记录的出差行程
2. 出差日期覆盖今天
3. 有包裹今日到达驿站

**推送内容**：
```
你出差期间有包裹到了：

📦 [商品名]（[平台]）
已到：[驿站名称]
超时时间：[日期 时间]

你 [回程日期] 才回来，sincerely建议：
· 让朋友/家人帮取
· 联系驿站说明情况延长保管

操作：[找人代取] [回来再取] [不用管]
```

| 操作 | 处理 |
|------|------|
| 找人代取 | 询问代取人手机号，发送取件码（mock） |
| 回来再取 | 计算回程日与超时日差值，若来得及则不再提醒；来不及则建议代取 |
| 不用管 | 标记跳过 |

---

## 6. 签收确认

当用户说"已经取了""签收了"或物流状态自动更新为 `signed` 时：

1. 将包裹状态更新为已签收
2. 从待追踪列表移除
3. 轻量确认："好的，[商品名] 已标记签收~"

---

## 7. 边界处理速查

| 场景 | 处理 |
|------|------|
| 用户无在途包裹 | 回复"你目前没有在途的包裹，安心~" |
| 运单号查不到信息 | 提示可能尚未揽收或信息延迟，建议稍后再查 |
| 同一包裹重复提醒 | 24小时内不重复推送同一包裹的超时提醒 |
| 用户问退货物流 | 告知"退货物流由退换货助手跟踪，我帮你看的是买到手的包裹" |
| 包裹超时费用纠纷 | 不介入，建议联系驿站或快递公司 |
| 用户问寄快递 | 告知暂不支持寄件 |
