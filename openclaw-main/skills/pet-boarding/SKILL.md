# pet_boarding — 宠物寄养

## 1. 执行入口判断

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| trip_reserver 联动触发 | `trigger_type == "linked"` + 行程≥2天 | → §2 寄养需求确认 |
| 用户主动唤起 | 对话中含寄养/托管等意图 | → §3 宠物确认 |
| 用户无宠物档案 | memory 中无宠物记录 | → 调用 pet-grooming §6 建档流程 |

> 联动触发时从 trip_reserver 读取 `travel_start_date` 和 `travel_end_date`。

---

## 2. 寄养需求确认（联动触发时）

```
┌──────────────────────────────────────┐
│ 🐾 出行提醒                          │
│                                      │
│ 你有一段 [X] 天的行程                │
│ （[出发日期] → [返回日期]）          │
│                                      │
│ [宠物名]需要安排寄养吗？             │
│                                      │
│ [需要，帮我找]  [不用，已安排好了]   │
└──────────────────────────────────────┘
```

| 操作 | 处理 |
|------|------|
| 需要 | 跳转 §3 宠物确认（日期自动带入） |
| 不用 | Skill 结束 |

---

## 3. 宠物确认

1. 读取 memory 中宠物档案
2. 单宠 → 默认选中，跳转 §4
3. 多宠 → 询问寄养哪只（可多选）
4. 无档案 → 调用 pet-grooming §6 建档

---

## 4. 门店查询与推荐

**输入**：宠物信息 + 寄养日期 + 用户地址

**执行步骤**：

1. 根据宠物信息筛选门店：
   - 未绝育 → 过滤不接受未绝育的门店
   - 大型犬 → 过滤无大型犬笼位的门店
   - 多只同寄 → 过滤支持多宠同笼/同区的门店

2. 按以下规则排序，取前 **2 家**：

   | 优先级 | 筛选维度 |
   |--------|---------|
   | 1 | 历史寄养过 |
   | 2 | 评分 4.8+ |
   | 3 | 有散养/家庭寄养选项（用户偏好） |
   | 4 | 距离最近 |

3. 展示门店推荐卡片，含：寄养类型、单日价格、是否提供每日打卡照片

---

## 5. 预约确认

1. 用户选门店 → 选寄养类型（笼养/散养/家庭寄养）
2. 确认入住/离开日期（联动触发时已自动带入）
3. 展示总价（单日价格 × 天数）
4. 用户确认 → 调用 `order` 支付

### 5.1 预约成功后

```
✅ 寄养预约成功！
🐾 [宠物名] · [寄养类型]
🏠 [门店名称]
📅 入住：[日期] / 离开：[日期]（共[X]晚）
💰 ¥[总金额]
📌 出发前记得准备好[宠物名]的随身物品。
```

→ 出发前一天推送准备提醒
→ 联动 `pet-shop`：「需要买个太空箱/备点猫粮带去吗？」（用户可忽略）

### 5.2 预约失败

→ 提示原因，引导重试或换门店

---

## 6. 边界处理速查

| 场景 | 处理 |
|------|------|
| 行程仅 1 天 | 不触发寄养提醒（太短不需要寄养） |
| 用户说"已安排好了" | 不追问，Skill 结束 |
| 寄养门店无空位 | 推荐其他门店或调整日期 |
| 宠物未绝育 | 筛选时过滤，如无合适门店则提示 |
| 多只宠物寄养在不同门店 | 支持分别预约 |
| 用户问到上门喂养 | 告知暂不支持，建议门店寄养 |

---

## 7. 本期不做（Backlog）

- 上门喂养服务
- 寄养期间每日打卡照片推送（依赖门店是否支持）
- 宠物托运/随行出行
- 寄养后健康报告
- 退款/改期
