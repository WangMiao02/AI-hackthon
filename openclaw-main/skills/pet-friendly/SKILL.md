# pet_friendly — 宠物友好场所推荐

## 1. 执行入口判断

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 用户主动询问 | 含带宠物/宠物友好/能带猫狗等意图 | -> S2 场所类型识别 |
| 用户无宠物档案 | memory 中无宠物记录 | -> 调用 pet-grooming S6 建档 |

---

## 2. 场所类型识别

| 用户意图 | 示例表达 | 场所类型 | 跳转 |
|---------|---------|---------|------|
| 吃饭 | "带狗去哪吃""宠物友好餐厅" | restaurant | -> S3 |
| 喝咖啡/下午茶 | "能带猫的咖啡店" | cafe | -> S3 |

## 3. 场所搜索与推荐

**输入**：场所类型 + 宠物信息（体型关键） + 用户地址

**执行步骤**：

1. 读取宠物档案中的体型信息（很多场所限制大型犬）
2. 搜索附近有「宠物友好」标签的场所
3. 按以下规则排序，取前 **3 个** 展示：

   | 优先级 | 筛选维度 |
   |--------|---------|
   | 1 | 明确标注宠物友好且接受该体型 |
   | 2 | 评分最高 |
   | 3 | 距离最近 |
   | 4 | 有用户晒宠物照片的评论（侧面验证） |

4. 展示推荐卡片：

```
为你和[宠物名]找到以下[场所类型]

1. [商户名称]
   评分[X]分 | 距你[X]km
   宠物规则：[具体规则]
   人均：[价格]元

2. [商户名称]
   评分[X]分 | 距你[X]km
   宠物规则：[具体规则]
   人均：[价格]元

操作：[选第1家] [选第2家] [看更多] [帮我叫车去]
```

**宠物规则字段示例**：
- "可带宠物入内，需牵绳"
- "仅限小型犬/猫，需装笼或使用宠物推车"
- "有户外宠物专区，室内不可带入"
- "不限体型，提供宠物饮水碗"

---

## 4. 用户选择后联动

| 场所类型 | 用户操作 | 联动 |
|---------|---------|------|
| restaurant | 选了餐厅 | 询问是否需要订座 -> `restaurant_reservation` |
| hotel | 选了酒店 | 询问入住日期 -> `hotel_reserver` |
| cafe/mall/park | 选了地方 | 询问是否需要叫车 -> `ride_hailing` |

---

## 5. 边界处理速查

| 场景 | 处理 |
|------|------|
| 附近无宠物友好场所 | 扩大搜索范围，或建议其他类型 |
| 宠物体型超出场所限制 | 标注不适合，推荐其他接受大型犬的场所 |
| 场所宠物规则不明确 | 标注"建议提前电话确认"，附电话 |
| 用户未带宠物档案（体型未知） | 直接展示所有结果，标注体型限制供参考 |
| 用户问到宠物乐园/游泳馆 | 告知属于宠物专属服务场所，非本 Skill 范围 |
| 节假日场所规则可能变化 | 提示"节假日规则可能调整，建议出发前确认" |
