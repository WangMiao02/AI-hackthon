# pet_grooming — 宠物美容洗澡

## 1. 执行入口判断

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时系统触发（周期复约） | `trigger_type == "scheduled"` | -> S5 周期复约提醒 |
| 用户主动唤起 | 对话中含宠物洗澡/美容/剪毛等意图 | -> S2 宠物确认 |
| 用户无宠物档案 | memory 中无宠物记录 | -> S6 宠物建档 |

---

## 2. 宠物确认

1. 读取 memory 中的宠物档案
2. 如果只有一只宠物 -> 默认选中，跳转 S3
3. 如果有多只宠物 -> 询问「这次是给哪位毛孩子洗澡？」列出宠物名字供选择
4. 如果无档案 -> 跳转 S6 宠物建档

---

## 3. 服务查询与推荐

**输入**：宠物信息（品种/体型）+ 用户地址

**执行步骤**：

1. 根据宠物体型过滤门店（部分门店不接受大型犬）
2. 根据宠物特殊需求过滤（如皮肤敏感则优先有药浴服务的门店）
3. 按以下规则排序，取前 **2 家** 展示：

   | 优先级 | 筛选维度 |
   |--------|---------|
   | 1 | 历史消费过该门店 |
   | 2 | 评分 4.8 分及以上 |
   | 3 | 距离最近 |
   | 4 | 价格（同体型对比） |

4. 无可用门店 -> 提示扩大搜索范围
5. 有结果 -> 展示门店推荐卡片，跳转 S4

---

## 4. 预约确认

**执行步骤**：

1. 用户选门店 -> 展示该店可选项目（洗澡/洗澡+剪毛/全套美容/药浴）
2. 项目按宠物体型自动匹配价格（小型/中型/大型犬猫价格不同）
3. 用户选项目 -> 展示可选时段
4. 用户选时段 -> 展示预约确认卡片
5. 用户确认 -> 调用 `order` 完成支付

### 4.1 预约结果处理

**成功**：
```
预约成功！
宠物：[宠物名] | 项目：[项目名称]
门店：[门店名称]
时间：[预约日期] [时间段]
地址：[门店地址]
金额：[金额]元
记得带上[宠物名]准时到店哦
```
-> 记录本次美容时间到 memory
-> 如用户未设置周期提醒，轻量询问：「要不要帮你设置下次洗澡提醒？」

**失败**：
-> 提示原因，引导重试或换门店/时段

---

## 5. 周期复约提醒

### 5.1 首次设置

用户说「帮我设置洗澡提醒」或预约成功后主动询问时触发：

```
多久提醒[宠物名]洗一次澡？
选项：[2周后]  [3周后]  [4周后]  [自定义]
```

记录：宠物名、服务类型、上次门店、周期、下次提醒日。

### 5.2 到期提醒推送

```
洗澡提醒
[宠物名]上次洗澡是 X 周前，该洗香香啦～

上次信息：
门店：[门店名称]
项目：[项目名称] | [价格]元

操作：[一键复约] [换个门店] [下次再说] [取消提醒]
```

| 操作 | 处理 |
|------|------|
| 一键复约 | 用上次门店，直接进时段选择 |
| 换个门店 | 跳转 S3 重新推荐 |
| 下次再说 | 推迟 3 天再提醒一次，第二次仍不操作则静默 |
| 取消提醒 | 删除周期任务 |

---

## 6. 宠物建档（首次使用）

渐进式收集，只问当前服务必需信息：

```
我还不认识你的毛孩子，简单介绍一下？
1. 叫什么名字？
2. 猫还是狗？（或其他）
3. 什么品种？
4. 体型大概多大？（小型/中型/大型）
```

用户可以一句话回答，如"我家豆豆，布偶猫"，自动解析并存入 memory。

> 补充信息（皮肤敏感、怕生等）不主动问，在后续交互中提到时自动记录。

---

## 7. 边界处理速查

| 场景 | 处理 |
|------|------|
| 门店不接受该体型 | 过滤掉，不展示 |
| 宠物有皮肤问题 | 优先推荐有药浴服务的门店 |
| 时段已被预约 | 提示换时段 |
| 支付失败 | 保留信息可重试 |
| 用户问到上门洗护 | 告知暂不支持，建议到店 |
| 周期提醒到期但近期已有预约 | 静默跳过 |
