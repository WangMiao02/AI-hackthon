# pet_shop — 宠物用品即时购买

## 1. 执行入口判断

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 用户主动购买 | 含买/需要/没了+宠物用品关键词 | -> S2 需求解析 |
| 其他 Skill 联动 | 传入明确品类（太空箱/处方粮等） | -> S3 商品搜索 |
| 新宠到家 | "刚养了只猫/狗""新来了一只" | -> S4 新宠采购清单 |
| 用户无宠物档案 | memory 中无宠物记录 | -> 调用 pet-grooming S6 建档 |

---

## 2. 需求解析

1. 从用户表达中提取：要买什么品类 + 有无品牌偏好 + 有无预算限制
2. 读取 memory 中宠物档案（品种/体型/年龄/健康状况）
3. 如果品类明确 -> 跳转 S3
4. 如果品类模糊（如"买点宠物的东西"）-> 询问具体需要什么

---

## 3. 商品搜索与推荐

**输入**：品类 + 宠物信息 + 用户地址

**执行步骤**：

1. 调用闪购接口搜索附近有货的商品
2. 根据宠物档案智能过滤：
   - 猫粮/狗粮 -> 按年龄过滤（幼猫粮/成猫粮/老年猫粮）
   - 猫砂盆 -> 按猫数量推荐尺寸（多猫推荐大号）
   - 航空箱 -> 按体型推荐尺寸
3. 按以下规则排序，取前 **3 个** 展示：

   | 优先级 | 筛选维度 |
   |--------|---------|
   | 1 | 历史购买过（复购优先） |
   | 2 | 销量最高 |
   | 3 | 评分最高 |
   | 4 | 配送最快 |

4. 展示商品推荐卡片：

```
为[宠物名]找到以下[品类]

1. [商品名称]
   [价格]元 | [店铺名] | 预计[X]分钟送达
   [月销XXX]

2. [商品名称]
   [价格]元 | [店铺名] | 预计[X]分钟送达
   [月销XXX]

3. [商品名称]
   [价格]元 | [店铺名] | 预计[X]分钟送达

操作：[买第1个] [买第2个] [买第3个] [看更多]
```

5. 用户选品 -> 调用 `order` 下单

---

## 4. 新宠采购清单

当检测到用户新养了宠物时触发：

1. 引导建档（如果还没有档案）
2. 根据宠物类型生成推荐采购清单：

**猫咪新手清单**：
- 猫粮（按年龄推荐）
- 猫砂 + 猫砂盆
- 食碗 + 水碗/饮水机
- 猫抓板
- 航空箱/太空舱

**狗狗新手清单**：
- 狗粮（按年龄体型推荐）
- 食碗 + 水碗
- 牵引绳 + 项圈
- 狗窝/垫子
- 拾便袋

3. 用户勾选需要的项目 -> 逐项搜索推荐 -> 确认后批量下单

---

## 5. 购买后联动

下单成功后：
- 如果是消耗品（粮/砂/零食）-> 询问「要加入定期补买提醒吗？」-> 联动 `pet-supplies`
- 如果不是消耗品 -> 不追问，Skill 结束

---

## 6. 边界处理速查

| 场景 | 处理 |
|------|------|
| 附近无货 | 扩大搜索范围，或推荐次日达商品 |
| 用户无明确品牌偏好 | 按销量+评分推荐 |
| 宠物档案不完整影响推荐 | 直接展示通用结果，不强制补档 |
| 用户要买处方药 | 告知需医生开具，建议到宠物医院购买 |
| 用户要买活体 | 明确告知不支持 |
