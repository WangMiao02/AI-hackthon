# pet_supplies — 宠物消耗品补买提醒

## 1. 执行入口判断

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时推送（到期提醒） | `trigger_type == "scheduled"` | -> S3 推送提醒卡片 |
| 用户主动唤起 | 含猫粮没了/猫砂该买/宠物补货等意图 | -> S2 前置检查 |
| 用户未完成配置 | 消耗品清单为空 | -> S4 Onboarding |
| 其他 Skill 联动 | pet-medical 传入新品类 | -> S4.2 添加单品 |

---

## 2. 前置检查

1. 读取 memory 中宠物消耗品清单
2. 清单为空 -> 跳转 S4 Onboarding
3. 清单存在 -> 计算各品类剩余天数，跳转 S3

---

## 3. 推送提醒卡片

```
[宠物名]的东西快用完啦

猫粮（渴望六种鱼 5.4kg）
预计 6月15日 用完 | 还剩 3 天

猫砂（豆腐猫砂 10L）
预计 6月18日 用完 | 还剩 6 天

操作：[一键复购同款] [这次想换一个] [还够用，延后提醒]
```

| 操作 | 处理 |
|------|------|
| 一键复购同款 | 传入上次商品信息，调用 `order` 下单 |
| 这次想换一个 | 转交 `pet-shop`，进入选品流程 |
| 还够用，延后提醒 | 推迟 3 天再提醒，最多延后 2 次 |

---

## 4. Onboarding（首次配置）

### 4.1 完整建档

```
帮[宠物名]建个补货清单吧～
先从最常买的开始，告诉我：
1. 买的什么？（如：渴望六种鱼猫粮 5.4kg）
2. 大概多久用完一袋/包？（如：30天）

可以一次说多个，比如"猫粮一个月一袋，猫砂两周一包"
```

每个品类记录：
- 品类名称 + 品牌规格
- 用户自估消耗周期
- 上次购买日期（从历史订单自动获取，或用户手动填）

### 4.2 添加单品（联动触发）

当 pet-medical 等 Skill 传入新品类时：

```
要把[品类名]加入定期补买清单吗？
大概多久用完一份？
选项：[1个月] [2个月] [3个月] [自定义]
```

---

## 5. 库存计算逻辑

```
预测耗尽日 = 上次购买日期 + 消耗周期
推送日 = 预测耗尽日 - 3天
```

**分档处理**：
- 固定消耗品（粮、砂）：精准倒计时，到期提醒
- 弹性消耗品（零食、营养品）：不单独提醒，在主粮提醒时附带询问「顺便补点零食吗？」

---

## 6. 边界处理速查

| 场景 | 处理 |
|------|------|
| 消耗品清单为空 | 触发 Onboarding |
| 库存充足（7天内不会耗尽） | 不推送 |
| 推送日已过（已耗尽） | 立即推送，标注"可能已经用完了" |
| 延后提醒超过 2 次 | 本轮静默，等下次购买后重新计算 |
| 用户在美团买了同款（订单同步） | 自动更新上次购买日期，重新计算周期 |
| 用户说"这个不用提醒了" | 从清单移除该品类 |

---

## 7. 本期不做（Backlog）

- 基于实际用量的智能周期修正（现阶段依赖用户自估）
- 多宠分别管理不同消耗品（现阶段按家庭统一管理）
- 跨平台订单同步（仅美团）
- 批量比价/凑单优化
- 弹性消耗品的独立提醒
