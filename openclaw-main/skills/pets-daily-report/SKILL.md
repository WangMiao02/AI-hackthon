# Skill: pet_daily_report

## 职责概述

每日22:00自动推送经营数据日报，汇聚所有 skill 写入的当日数据。三层折叠结构：今日（默认展开）/ 近7日 / 近30日。异常数据标🔴提示，不拦截操作。商家也可随时主动查询。

---

## §1 执行入口判断

| 触发来源 | 触发条件 | 进入分支 |
|---------|---------|---------|
| 系统定时 | 每日 22:00 | §3-A 自动日报生成与推送 |
| 商家主动 | 发送"日报"/"数据"/"今天怎么样"/"营收"等关键词 | §2 前置检查 → §3-B 主动查询 |

---

## §2 前置检查（商家主动触发时执行）

```
1. 读取当前时间：
   - 22:00前查询 → 展示今日截至当前实时数据，标注"数据截至 {当前时间}，营业中"
   - 22:00后查询 → 展示完整当日数据
2. 检查各数据源接口连通性：
   - 全部正常 → 正常生成日报
   - 部分异常 → 正常展示可获取数据，异常项标注"数据暂时无法获取"
```

---

## §3 核心逻辑

### §3-A 自动日报生成与推送

```
触发：每日 22:00 定时

1. 从各 skill 数据写入记录中汇聚今日数据：
   - appointment_manager：订单量 / 完成量 / 爽约量 / 取消量
   - pet_review_manager：好评数 / 差评数 / 高危差评数 / 回复率
   - pet_promotion_advisor：当日促销订单占比（若有进行中促销）
   - 平台基础数据接口：营收 / 客单价 / 新客数 / 复购客数

2. 执行异常检测（见 §4 异常判断规则）
3. 生成日报卡片（见卡片A）
4. 静默推送，无需商家确认
```

### §3-B 主动查询

```
触发：商家主动发起

1. 读取实时数据（同 §3-A 数据汇聚逻辑）
2. 生成日报卡片（见卡片A），标注实时数据说明
3. 商家可点击展开近7日 / 近30日数据层
```

---

## §4 异常判断规则

| 指标 | 异常条件 | 标注方式 |
|------|---------|---------|
| 爽约率 | 当日爽约率 > 20% | 🔴 爽约率异常，今日 {X}% |
| 差评数 | 当日差评数 ≥ 3条 | 🔴 今日差评较多，共 {X} 条 |
| 高危差评 | 当日存在任意1条高危差评 | 🔴 今日有高危差评待处理 |
| 营收 | 较近7日日均下滑 ≥ 30% | 🔴 今日营收明显低于近期均值 |
| 完成订单量 | 较近7日日均下滑 ≥ 30% | 🔴 今日完成订单量明显偏低 |
| 取消率 | 当日取消率 > 15% | 🔴 今日取消率偏高，共 {X} 单取消 |

> 异常标注只展示，不推额外卡片，不触发其他 skill，商家自行决定是否处理。

## §6 数据接口依赖

| 数据项 | 来源 | 读/写 |
|--------|------|------|

## §7 边界处理速查

| 场景 | 处理方式 |
|------|---------|
| 当日无任何订单（休息日/零单日） | 正常推送日报，所有数据显示0，不推异常标注 |
| 22:00前商家主动查询 | 展示截至当前实时数据，标注"营业中，数据持续更新" |
| 部分数据接口异常 | 展示可获取数据，异常项显示"暂时无法获取"，不阻断整体推送 |
| 商家连续7日未打开日报 | 降频为每3日推送一次，商家主动查询时恢复每日推送 |
| 当日有高危差评未处理 | 日报🔴标注提示，不重复推 pet_review_manager 的高危卡片 |
| 促销数据为空（无进行中促销） | 隐藏促销模块，不展示空数据块 |
