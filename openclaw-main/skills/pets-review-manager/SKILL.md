# Skill: pet_review_manager

## 职责概述

管理宠物美容店的全部顾客评价：好评自动回复（活泼可爱口吻，静默执行）、差评三级分类处理（高危拦截 / 普通推草稿确认 / 中性自动回复）。高危差评（宠物受伤/生病/死亡）禁止 AI 自动回复，强制推处理指引卡片由商家主导处置。

---

## §1 执行入口判断

| 触发来源 | 触发条件 | 进入分支 |
|---------|---------|---------|
| 平台事件 | 新评价到达（任意星级） | §3-A 评价分级路由 |
| 商家主动 | 发送"评价"/"差评"/"回复"等关键词 | §2 前置检查 → §3-E 主动查询 |
| 商家主动 | 点击草稿卡片"修改后发送"按钮 | §3-D 草稿编辑与发送 |

---

## §2 前置检查（商家主动触发时执行）

```
1. 读取近7日未处理评价列表
2. 检查是否有高危差评待处理 → 有则优先展示
3. 检查数据接口连通性 → 异常时提示"评价数据加载失败，请稍后重试"
```

---

## §3 核心逻辑

### §3-A 评价分级路由

```
触发：新评价到达事件

1. 读取评价内容 + 星级
2. 执行高危关键词检测（优先级最高）：
   - 命中高危关键词 → 立即进入 §3-B 高危处理，终止后续流程
3. 未命中高危 → 按星级分级：
   - 4–5星 → §3-C 好评自动回复
   - 1–2星，或3星含明确投诉词 → §3-D 普通差评草稿
   - 3星无明确投诉词 → §3-C 中性自动回复
```

**高危关键词列表（持续维护）**

| 类别 | 关键词示例 |
|------|---------|
| 宠物受伤 | 受伤、划伤、流血、骨折、受到伤害、弄伤、夹伤 |
| 宠物生病 | 生病、发烧、感染、过敏、皮肤病、呕吐、精神差 |
| 宠物死亡 | 死了、去世、没了、死亡、离开了、没救回来 |
| 明确追责 | 赔偿、投诉、律师、报警、曝光、12315 |

> 关键词匹配采用模糊匹配，避免"没有受伤"误判为高危；上下文否定句自动降级为普通差评。

---

### §3-B 高危差评处理

```
触发：评价命中高危关键词

1. 立即冻结该评价的自动回复权限（本 skill 不得生成任何回复草稿）
2. 推高危差评处理指引卡片（见卡片A）
3. 写入高危差评记录，供 pet_daily_report 标注
4. 等待商家操作，不超时催促，不降频
```

---

### §3-C 自动回复（好评 + 中性）

```
触发：4–5星好评 / 3星无明确投诉词

1. 读取评价内容，提取关键信息：
   - 宠物名称（如顾客提及）
   - 夸赞的服务项目（如"洗澡很干净"/"师傅很温柔"）
2. 生成活泼可爱口吻回复（见回复模板）
3. 静默发送，不推通知卡片
4. 写入回复记录，供 pet_daily_report 统计好评回复率
```

**好评回复模板（活泼可爱口吻）**

| 场景 | 回复示例 |
|------|---------|
| 提及宠物名称 | "谢谢你带{宠物名}来找我们～能得到你们的认可真的超开心！期待{宠物名}下次来玩儿🐾" |
| 夸赞师傅 | "哇谢谢你的夸奖！师傅们看到一定会很开心的😊 下次见～" |
| 夸赞整体服务 | "谢谢你的好评！你和毛孩子的满意就是我们最大的动力，随时欢迎回来🐶✨" |
| 中性3星无投诉 | "感谢你抽时间留下评价！如果有任何不满意的地方欢迎告诉我们，我们会努力做得更好～" |

> 回复字数控制在 40–80 字，不使用"尊敬的顾客"等生硬措辞，emoji 适量（1–2个）。

---

### §3-D 普通差评草稿处理

```
触发：1–2星评价 / 3星含明确投诉词

1. 读取评价内容，识别投诉类型：
   - 服务态度类（师傅态度差/不耐烦）
   - 效果类（剪得不好看/洗得不干净）
   - 等待类（等太久/时间不准时）
   - 价格类（觉得贵/不值）
   - 其他类

2. 根据投诉类型生成差评回复草稿（见草稿模板）
3. 推差评草稿确认卡片（见卡片B），等待商家操作
4. 商家点击"确认发送" → 直接发送草稿内容
5. 商家点击"修改后发送" → 进入 §3-D 草稿编辑
6. 商家点击"暂不回复" → 记录忽略，24小时后重新推送提醒
```

**差评回复草稿模板**

| 投诉类型 | 草稿示例 |
|---------|---------|
| 服务态度类 | "非常抱歉这次的体验让你不开心😔 我们已经和师傅沟通过了，态度问题我们零容忍。欢迎再给我们一次机会，下次一定让你满意！" |
| 效果类 | "非常抱歉效果没有达到你的期望！下次来店可以提前和师傅说明你的想法，我们会尽力做到你满意的样子🐾" |
| 等待类 | "非常抱歉这次让你久等了！我们会优化预约安排，尽量减少等待时间。感谢你的理解和耐心～" |
| 价格类 | "感谢你的反馈！我们的定价包含了{服务项目}，如有疑问欢迎随时联系我们，我们很乐意详细说明😊" |
| 其他类 | "非常抱歉这次的体验没有达到你的期望😔 你的反馈对我们很重要，我们会认真改进。欢迎再次光临！" |

> 草稿字数控制在 50–100 字；不承认具体过失（法律风险），不做具体赔偿承诺。

## §4 卡片设计

### 卡片A — 高危差评处理指引
```
🔴 收到高危差评，需要你来处理

顾客：{顾客昵称}
评价内容："{评价摘要，不超过50字}"
预警原因：{命中关键词，如"宠物受伤"}

⚠️ 此类评价涉及宠物安全，AI 不会自动回复
请选择下一步处理方式：

[📞 联系顾客]  [📋 发起申诉]  [💬 联系平台客服]
```
> 联系顾客：跳转顾客IM对话
> 发起申诉：跳转 pet_review_appeal skill
> 联系平台客服：跳转平台官方客服入口

```

"{回复草稿全文}"
```

## §6 边界处理速查

| 场景 | 处理方式 |
|------|---------|
| 高危关键词误判（否定句） | 上下文含"没有""不是""并没有"等否定词 → 自动降级为普通差评，不推高危卡片 |
| 评价内容为空（仅有星级） | 4–5星空评 → 用通用好评模板静默回复；1–2星空评 → 推草稿卡片，草稿用通用差评模板 |
| 商家连续3次忽略差评草稿推送 | 降频为每日汇总推送一次，不逐条打扰 |
| 平台回复接口失败 | 推错误提示，建议商家手动回复，不重试 |
| 同一顾客同日多条评价 | 逐条独立处理，不合并 |
| 顾客追评（修改评价） | 视为新评价重新触发分级路由 |
| 高危差评商家长时间未处理（超72小时） | 推一次温和提醒卡片，之后不再催促 |
