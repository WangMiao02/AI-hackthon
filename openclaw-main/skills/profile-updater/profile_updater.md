# profile_updater — 用户画像迭代

## 1. 执行入口判断

**首先判断触发来源，然后跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时系统触发 | `trigger_type == "scheduled"`（每日 22:00） | → §2 会话总结 |
| 其他来源 | 任何非定时触发 | → 拒绝执行，Skill 结束 |

> 本 Skill **不接受用户主动唤起**，也不接受其他 Skill 联动触发。

---

## 2. 会话总结（静默）

1. 读取当天所有 agent 会话记录
2. 如果**当天无有效会话** → Skill 结束，不写入任何更新
3. 如果**有有效会话** → 提炼以下信号，跳转 §3 画像更新

**提炼维度**：

| 类别 | 分析信号 |
|------|---------|
| 出行类 | 出行频率变化、交通方式选择、惯常出发时段 |
| 住宿类 | 价位选择趋势、房型偏好变化 |
| 饮食类 | 菜系 / 口味偏好变化、常点餐厅、用餐时段规律 |
| 消费类 | 单次消费价位区间、对优惠券的响应率 |
| 健康类 | 用药周期规律、补药行为模式 |
| 家庭类 | 日用品消耗速度、品类购买规律 |
| 行为习惯类 | 活跃时段、推送响应率（哪类推送常点 / 常忽略）、从看到推荐到下单的决策速度 |

---

## 3. 画像更新（静默）

**字段隔离规则**（硬性约束，不可绕过）：

```
用户手填字段（user_input）   →  只读，永不覆盖
agent分析字段（agent_inferred）  →  可读写，本 Skill 仅操作此类
```

**执行步骤**：

1. 对比新分析结果与当前 `agent_inferred` 字段值
2. 如果**无变化** → 不写入，Skill 结束
3. 如果**有变化** → 仅更新发生变化的字段，其余保持不变
4. 将更新内容同步至对应 Skill 的 Onboarding 配置（`agent_inferred` 分区）
5. 写入更新日志（供内部调试，用户不可见），Skill 结束

**各 Skill 更新映射**：

| Skill | 更新的 agent_inferred 字段示例 |
|-------|-------------------------------|
| `trip_reserver` | 惯常出发时段、交通方式倾向 |
| `hotel_reserver` | 实际价位选择区间、房型选择规律 |
| `food_recommend` | 菜系 / 口味偏好权重、常点时段 |
| `expense_reimbursement` | 报销周期规律 |
| `coupon_claim` | 券类响应率、偏好券种 |
| `medicine_reminder` | 实际补药周期 |
| `supplies_reminder` | 实际消耗速度修正值 |

---

## 4. 边界处理速查

| 场景 | 处理 |
|------|------|
| 当天无有效会话 | 不写入任何更新，静默结束 |
| 分析结果与现有值无差异 | 不写入，静默结束 |
| 检测到用户手填字段 | 跳过，严格只操作 agent_inferred 分区 |
| 会话数据读取失败 | 记录错误日志，静默结束，次日重新执行 |
| 单一信号样本过少（如只有1次会话）| 保守更新，不覆盖已有值，累积后再更新 |

---

## 5. 本期不做（Backlog）

> 以下功能**不在当前 Skill 范围内**：

- 向用户展示当前画像内容
- 用户手动修正 agent 分析结果
- 跨用户画像对比 / 群体偏好分析
- 画像版本回滚（误判时恢复上一版本）
