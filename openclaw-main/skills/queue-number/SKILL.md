# queue_number — 餐厅排队取号

## 1. 执行入口判断

**首先判断触发来源，然后跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| food_recommend 联动触发 | 用户选择餐厅后未下外卖单 + 检测到堂食意图信号 | → §2 堂食意图确认 |
| 用户主动唤起 | 对话中含堂食/取号/排队/等位等意图 | → §2 堂食意图确认 |

**堂食意图信号**（以下任一满足即触发）：
- 用户查看了餐厅地址 / 导航入口
- 用户查看了餐厅详情页停留超过一定时长
- 用户在对话中提及「去吃」「堂食」「过去」等关键词

---

## 2. 堂食意图确认

推送询问卡片，等待用户明确回复：

```
┌──────────────────────────────────────┐
│ 🍽 你是打算去老乡鸡堂食吗？          │
│ 检测到你在查看餐厅地址               │
│                                      │
│    [是，帮我盯着]     [不用了]       │
└──────────────────────────────────────┘
```

**用户响应分支**：
- 点击 `是，帮我盯着` → §3 排队状态监听
- 点击 `不用了` → Skill 结束，不再推送
- 无响应超时（5分钟）→ Skill 结束，不打扰用户

---

## 3. 排队状态监听（后台静默）

用户确认后，后台持续轮询目标餐厅排队状态，用户无感知：

1. 读取美团餐厅实时排队状态
2. 检查是否支持线上取号

**状态判断分支**：

| 状态 | 处理 |
|------|------|
| 当前无需排队（可直接入座） | 推送提示「目前无需排队，可直接前往」，Skill 结束 |
| 当前需要排队 + 支持线上取号 | 立即跳转 §4 自动取号 |
| 收到线上取号开放通知 | 立即跳转 §4 自动取号 |
| 不支持线上取号 | 推送提示「该餐厅暂不支持线上取号，请到店自行取号」，Skill 结束 |

> 监听超时限制：用户确认后 **2小时内** 未触发取号条件 → 推送提示「已停止监听，如需取号请重新发起」，Skill 结束

---

## 4. 自动取号

1. 调用美团线上取号接口，自动为用户取号

**取号成功**：
```
┌──────────────────────────────────────┐
│ ✅ 已帮你取到号！                    │
│                                      │
│ 老乡鸡（静安寺店）                   │
│ 你的号码：A23                        │
│ 前面还有：8 桌                       │
│ 预计等待：约 40 分钟                 │
│                                      │
│ 📌 轮到你时我会再提醒你              │
└──────────────────────────────────────┘
```
→ 继续监听排队进度，轮到用户时推送「快到你了，准备入座！」，Skill 结束

**取号失败**：
```
⚠️ 自动取号失败
名额已满 / 接口异常，请到店自行取号
```
→ Skill 结束

---

## 5. 边界处理速查

| 场景 | 处理 |
|------|------|
| 用户未响应确认卡片（超时5分钟） | 静默结束，不打扰 |
| 餐厅不支持线上取号 | 提示到店自行取号，Skill 结束 |
| 当前无需排队 | 推送提示可直接前往，Skill 结束 |
| 监听超过2小时未触发 | 推送提示已停止监听，Skill 结束 |
| 取号时名额已满 | 提示失败原因，引导到店自行取号 |
| 用户已到店但未取到号 | 同上 |

---

## 6. 本期不做（Backlog）

> 以下功能**不在当前 Skill 范围内**，遇到请告知用户暂不支持：

- 电话预约 / 人工排队场景
- 多人就餐人数选择（影响等位时长估算）
- 跨餐厅同时监听多个排队状态
- 取号后的改签 / 取消排队
- 非美团平台餐厅取号
