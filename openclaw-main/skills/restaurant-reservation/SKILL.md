# restaurant_reservation — 餐厅堂食预约

## 1. 执行入口判断

**首先判断触发来源，跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| food_recommend 联动触发 | 用户选择餐厅后检测到堂食+订座意图 | → §3 预约能力探测（目标餐厅已知） |
| 用户主动唤起（有目标餐厅） | 对话中含具体餐厅名 + 订座/聚餐意图 | → §3 预约能力探测 |
| 用户主动唤起（无目标餐厅） | 对话中含聚餐/找餐厅/订位等意图，无具体餐厅 | → §2 用餐信息收集 |
| 用户未完成配置 | 首次使用，常用地址未设置 | → §7 Onboarding |

---

## 2. 用餐信息收集

用户无目标餐厅时，先收集基本信息再推荐：

```
🍽 帮你找餐厅～先告诉我几个信息：

用餐时间：[今晚]  [明天]  [自定义日期]
用餐人数：[2人]  [4人]  [6人以上]  [自定义]
偏好菜系：[不限]  [中餐]  [日料]  [火锅]  [西餐]  [更多]
```

收集完成后 → §4 餐厅推荐

> 用户若直接说「明晚 4 个人吃火锅」，自动解析时间/人数/菜系，无需逐项追问，直接跳转 §4。

---

## 3. 预约能力探测

**目标餐厅已知时，立即检测该餐厅的预约能力：**

| 探测结果 | 处理 |
|---------|------|
| ✅ 支持在线按钮预约 | → §5 在线订座流程 |
| 📞 仅支持电话预约 | → 推送电话预约提示卡片（含电话号码），Skill 结束 |
| 🔢 不支持订座，支持线上取号 | → 自动转交 `queue_number` skill |
| ❌ 不支持任何预约方式 | → 提示「该餐厅暂不支持预约，建议直接前往」，Skill 结束 |

**电话预约提示卡片**：
```
┌──────────────────────────────────────┐
│ 📞 该餐厅仅支持电话预约              │
│                                      │
│ 🍜 [餐厅名称]                        │
│ 📍 [餐厅地址]                        │
│ 📞 预约电话：[电话号码]              │
│ ⏰ 接听时间：[营业时间]              │
│                                      │
│ 💡 建议提前 1 天致电预约             │
│                                      │
│ [拨打电话]  [直接前往]               │
└──────────────────────────────────────┘
```

## 4. 餐厅推荐

**输入**：用餐时间 + 人数 + 菜系偏好 + 用户地址（GPS/常用地址）

**筛选规则**：

| 优先级 | 筛选维度 | 说明 |
|--------|---------|------|
| 1 | 支持在线订座 | 优先推荐可直接按钮预约的餐厅 |
| 2 | 历史消费过该餐厅 | 熟悉口味，用户信任度高 |
| 3 | 菜系匹配 | 与用户偏好菜系一致 |
| 4 | 评分 | 4.5分及以上优先 |
| 5 | 距离 | 同等条件取距离最近 |

取前 **3 家** 展示，超出折叠；无结果 → 提示调整条件

**冷启动处理**（无历史消费记录）：按综合评分从高到低排列

推荐结果展示后 → 用户选择餐厅 → §3 预约能力探测

---

## 5. 在线订座流程

**执行步骤**：

1. 展示该餐厅可用时段（按用餐时间 + 人数过滤）
2. 用户选择时段 → 展示订座确认卡片
3. 用户确认 → 调用在线订座接口 → §5.1 预约结果处理

时段卡片及确认卡片格式详见 [references/reservation-flow.md](references/reservation-flow.md)

### 5.1 预约结果处理

**成功**：
```
✅ 订座成功！
🍽 [餐厅名称]
📅 [用餐日期] [时间段]
👥 [用餐人数] 人
📍 [餐厅地址]
📌 用餐前 1 小时会提醒你出发。
```
→ 联动 `ride_hailing`：推送「需要帮你叫车去餐厅吗？」

**失败**：
→ 提示失败原因（时段已满/接口异常），引导重新选择时段或换餐厅

---

## 6. 边界处理速查

| 场景 | 处理 |
|------|------|
| 餐厅仅支持电话预约 | 推送电话预约提示卡片（§3），提供电话号码 |
| 餐厅不支持订座但支持取号 | 自动转交 `queue_number` skill，告知用户「该餐厅不支持提前订座，已帮你切换到排队取号」 |
| 餐厅完全不支持预约 | 提示直接前往，Skill 结束 |
| 所选时段已满 | 提示「该时段已满」，展示其他可用时段 |
| 用餐人数超出餐厅单桌上限 | 提示「该餐厅单桌最多 X 人，如需大桌请致电餐厅」 |
| GPS 未授权 | 降级使用 Onboarding 中的常用地址 |
| 常用地址未设置 | 触发 §7 Onboarding |
| 用户已有该餐厅预约 | 提示「你已有该餐厅的预约」，询问是否修改 |

## 7. Onboarding（首次配置）

| 配置项 | 示例 | 说明 |
|--------|------|------|
| 常用地址 | 北京市朝阳区XX路XX号 | 默认搜索中心 |
| 菜系偏好（可选） | 中餐 / 日料 / 火锅 | 餐厅推荐参考 |
| 常用联系电话 | 138xxxxxxxx | 电话预约时使用 |

> 配置完成后自动跳转 §2 用餐信息收集。已配置用户可在「订座设置」中修改。
