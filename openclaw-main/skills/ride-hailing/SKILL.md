# ride_hailing — 一键打车

## 1. 执行入口判断

**首先判断触发来源，跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 用户主动唤起 | 对话中含打车/叫车/约车等意图 | → §2 主动叫车 |
| 天气智能触发 | `trigger_type == "scheduled"` + 恶劣天气检测通过 | → §3 天气推送 |
| 上下文联动触发 | `trigger_type == "linked"` + 上游传入目的地信号 | → §4 联动推送 |
| 用户未完成配置 | 常用地址缺失 | → §5 Onboarding |

> **当日免打扰检查**：§3 和 §4 执行前，先检查当日是否存在免打扰标记 `ride_hailing_muted`。若存在 → 静默跳过，Skill 结束。§2 用户主动触发时**忽略**此标记。

---

## 2. 主动叫车

1. 获取出发地（GPS 优先，降级常用地址）
2. 询问目的地（若上下文已有目的地则直接使用）
3. 查询可用车型与预估价格，按以下规则排序：

   | 优先级 | 维度 | 说明 |
   |--------|------|------|
   | 1 | 用户历史偏好车型 | 近期最常选的车型置顶 |
   | 2 | 预计等待时长 | 同等条件下优先等待更短的 |
   | 3 | 价格 | 同等条件下优先价格更低的 |

   **冷启动处理**（无历史用车记录）：默认展示快车/优享/专车三档，按价格从低到高排列

4. 推送车型选择卡片 → 用户确认 → 下单叫车

---

## 3. 天气智能触发

**出行时间推断**：

| 条件 | 处理 |
|------|------|
| 有 ≥5 条历史出行记录 | 取近 30 天工作日出行时间众数作为「习惯出行时间」 |
| 历史记录不足 | 降级使用默认窗口：工作日 7:30–9:00 |

在「习惯出行时间」前 **30 分钟** 调用天气接口，满足以下任一条件即推送：

| 天气类型 | 触发阈值 |
|---------|---------|
| 雨天 | 小雨及以上 |
| 高温 | 气温 ≥ 35°C |
| 大风 | 风力 ≥ 5 级 |
| 雾霾 | AQI ≥ 150 |
| 雪天 | 小雪及以上 |

推送打车建议卡片 → 用户点「一键叫车」进入 §2 确认下单（出发地/目的地预填）；点「不用了」→ 写入当日免打扰标记，Skill 结束。

---

## 4. 上下文联动触发

收到上游 Skill 传入的目的地信号后，满足以下**全部条件**才推送：

| 条件 | 说明 |
|------|------|
| 目的地距当前位置 > 2km | 太近不推荐打车 |
| 当日免打扰标记不存在 | 用户未选「不用了」 |
| 距出行时间 ≤ 60 分钟 | 太早推送无意义 |

**联动触发时机说明**：

| 上游 Skill | 触发时机 | 推送方式 |
|-----------|---------|---------|
| queue_number | 用户确认去某餐厅堂食（点击「是，帮我盯着」）时**立即**推送 | 即时推送 |
| concert_reserver / attraction_ticket / movie_ticket | 用户确认购票后推送 | 即时推送 |
| hotel_reserver | 用户确认预订酒店后推送 | 即时推送 |
| trip_reserver | 用户确认购买高铁/机票后，询问是否预约出发当天的接送车 | 预约叫车 |

**trip_reserver 联动专项逻辑（预约叫车）**：

**Step 1：订票成功后，先询问是否需要预约用车**

用户确认购票后，立即推送一句轻量询问，不做任何计算：
```
┌──────────────────────────────────────┐
│ 🚄 G1次  北京南站  6月15日 07:00 出发│
│                                      │
│ 要帮你提前预约去车站的车吗？         │
│                                      │
│    [要，帮我约]      [不用了]        │
└──────────────────────────────────────┘
```
- 用户点「不用了」→ 写入当日免打扰标记，Skill 结束
- 用户点「要，帮我约」→ Step 2

**Step 2：确认出发地**

检查是否已有常用家庭地址：

| 情况 | 处理 |
|------|------|
| 已有常用家庭地址 | 直接使用，跳过询问，进入 Step 3 |
| 无常用地址，有 GPS 定位 | 询问用户「从当前位置出发还是从家出发？」，若选家则请用户输入地址并保存 |
| 无常用地址，无 GPS | 请用户输入出发地址，询问是否保存为常用地址 |

**Step 3：计算行程时间，推送出发时间建议**

根据出发地到车站的实时路况预估行程时间：

```
建议出发时间 = 车次发车时间 - 打车行程时间 - 候车缓冲（30分钟）
```

| 打车行程时间 | 建议提前出发 | 是否需用户确认 |
|------------|------------|--------------|
| < 30 分钟  | 约 60 分钟前 | 否，直接展示 |
| 30–60 分钟 | 约 60–90 分钟前 | 否，直接展示 |
| > 60 分钟  | 超过 90 分钟前 | **是，需用户确认** |

推送出发时间确认卡片：
```
┌──────────────────────────────────────┐
│ 🚗 预约叫车                          │
│                                      │
│ 出发地：北京朝阳区XX小区             │
│ 目的地：北京南站                     │
│ 📍 预估行程：约 35 分钟              │
│ 建议出发：6月15日 06:00              │
│ 预估费用：¥45                        │
│                                      │
│  [就这个时间] [我想早点] [修改出发地]│
└──────────────────────────────────────┘
```

**Step 4：用户确认后完成预约**

```
┌──────────────────────────────────────┐
│ ✅ 预约成功！                        │
│                                      │
│ 6月15日 06:00 从北京朝阳区XX小区出发 │
│ 目的地：北京南站                     │
│ 预估费用：¥45                        │
│                                      │
│ 📌 出发前10分钟我会提醒你司机到了   │
└──────────────────────────────────────┘
```
- 到预约时间自动派单
- 派单成功后立即通知用户司机信息和预计到达时间
- 司机到达前 10 分钟再次推送提醒
- 用户点「到时候再说」→ 出发当天提前 90 分钟再次提醒

> 关键原则：只要用户**确认了出行目的地**，无论后续流程是否完成，ride_hailing 都应立即推送打车建议，不能等到取号/购票完成后才联动。

推送联动打车建议卡片示例：
```
┌──────────────────────────────────────┐
│ 🚗 要打车去晴山日本料理吗？          │
│                                      │
│ 📍 步行约 18 分钟                    │
│ 预估打车：¥18-25 · 等待约 3 分钟    │
│                                      │
│    [一键叫车]      [不用了]          │
└──────────────────────────────────────┘
```

用户点「一键叫车」→ 进入 §2 确认下单（目的地预填餐厅地址）；点「不用了」→ 写入当日免打扰标记，Skill 结束。

## 5. Onboarding（首次配置）

引导用户依次填写：

| 配置项 | 示例 |
|--------|------|
| 家（出发地常用地址） | 上海市长宁区XX路XX号 |
| 公司（目的地常用地址） | 上海市静安区南京西路XX号 |
| 惯用车型偏好 | 快车 / 优享 / 专车 / 无偏好 |

> 配置完成后自动跳转 §2 执行。已配置用户可在「打车设置」中修改。

---

## 6. 边界处理速查

| 场景 | 处理 |
|------|------|
| 当日已选「不用了」 | §3/§4 静默跳过；§2 主动触发不受影响 |
| GPS 未授权 | 降级使用常用地址，卡片注明「基于常用地址」 |
| 常用地址未设置 | 触发 §5 Onboarding |
| 附近无可用车辆 | 提示「当前附近暂无可用车辆，请稍后重试」 |
| 叫车失败 | 提示失败原因，保留目的地信息可重试 |
| 天气接口异常 | 跳过天气触发，不影响用户主动叫车 |

---

## 7. 本期不做（Backlog）

> 以下功能**不在当前 Skill 范围内**，遇到请告知用户暂不支持：

- 代驾 / 顺风车 / 拼车
- 多平台聚合比价（仅美团打车）
- 行程分享给联系人
- 发票开具
