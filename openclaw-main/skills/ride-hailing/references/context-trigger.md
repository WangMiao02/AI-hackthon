# 联动触发 — 各上游场景卡片模板

## 触发前置检查（所有场景通用）

| 检查项 | 通过条件 |
|--------|---------|
| 当日免打扰标记 | 不存在 `ride_hailing_muted` |
| 距离判断 | 目的地距当前位置 > 2km |
| 时间窗口 | 距出行时间 ≤ 60 分钟 |

三项全部通过才推送卡片，任一不满足则静默跳过。

---

## 场景一：排队取号后联动（来自 skill_queue_number）

```
┌─────────────────────────────────────────┐
│ 🎫 已为您取号：[餐厅名] 第 XX 号         │
│ ⏱ 预计等待：约 XX 分钟                  │
│                                         │
│ 🚗 需要打车过去吗？                      │
│ 📍 [餐厅地址]  距您约 X.X km            │
│ 💰 预估打车费：¥XX–XX                   │
├─────────────────────────────────────────┤
│  [一键叫车]          [不用了]             │
└─────────────────────────────────────────┘
```

**推送时机**：取号成功后立即推送（等待时间 > 10 分钟时才推送，太近不推荐打车）

## 场景二：演出/门票预订后联动（来自 concert_reserver / attraction_ticket / movie_ticket）

```
┌─────────────────────────────────────────┐
│ 🎵 [演出名称] 票已预订                   │
│ 📅 [演出日期时间] | 📍 [演出场馆]        │
│                                         │
│ 🚗 演出当天需要打车吗？                  │
│ 💰 预估打车费：¥XX–XX                   │
├─────────────────────────────────────────┤
│  [演出当天提醒我]    [不用了]             │
└─────────────────────────────────────────┘
```

**推送时机**：预订成功后立即推送询问；若用户选「演出当天提醒我」，则在演出日当天出行时间前 30 分钟再次推送叫车卡片

---

## 场景三：酒店预订后联动（来自 hotel_reserver）

```
┌─────────────────────────────────────────┐
│ 🏨 [酒店名称] 已预订                     │
│ 📅 入住：[日期] | 📍 [酒店地址]          │
│                                         │
│ 🚗 入住当天需要打车去酒店吗？             │
│ 💰 预估打车费：¥XX–XX                   │
├─────────────────────────────────────────┤
│  [入住当天提醒我]    [不用了]             │
└─────────────────────────────────────────┘
```

**推送时机**：预订成功后立即推送询问；若用户选「入住当天提醒我」，则在入住日当天根据历史出行习惯时间前 30 分钟推送叫车卡片

---

## 通用交互分支

| 用户操作 | 处理 |
|---------|------|
| 点击「一键叫车」 | 跳转 §2 主动叫车，目的地预填为上游传入地址 |
| 点击「不用了」 | 写入当日免打扰标记，Skill 结束 |
| 点击「XX当天提醒我」 | 记录提醒任务，在指定日期重新触发本 Skill |
| 上游未传入目的地 | 静默跳过，不推送联动卡片 |
