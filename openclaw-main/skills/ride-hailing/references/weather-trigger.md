# 天气触发 — 卡片模板与免打扰机制

## 天气推送卡片

```
┌─────────────────────────────────────────┐
│ ☔ 今天[天气描述]，建议打车出行            │
│ 🌡 [气温] | 💨 [风力] | 🌫 AQI [数值]   │
│                                         │
│ 📍 出发：[常用出发地]                    │
│ 📍 目的地：[常用目的地]                  │
│ ⏱ 预计行程：[时长] | 💰 预估：¥XX–XX    │
├─────────────────────────────────────────┤
│  [一键叫车]          [不用了]             │
└─────────────────────────────────────────┘
```

**天气描述文案参考：**

| 天气类型 | 卡片标题文案 |
|---------|------------|
| 雨天 | ☔ 今天有雨，建议打车出行 |
| 高温 | 🌡 今天高温 XX°C，建议打车出行 |
| 大风 | 💨 今天风力 X 级，建议打车出行 |
| 雾霾 | 🌫 今天空气质量较差（AQI XX），建议打车出行 |
| 雪天 | ❄️ 今天有雪，路面湿滑，建议打车出行 |
| 多条件叠加 | 取最严重的一条作为主标题，其余在天气行展示 |

## 免打扰机制

| 用户操作 | 处理 |
|---------|------|
| 点击「一键叫车」 | 跳转 §2 主动叫车，出发地/目的地预填 |
| 点击「不用了」 | 写入当日免打扰标记 `ride_hailing_muted`，Skill 结束 |
| 当日已有免打扰标记 | §3 天气触发静默跳过，不推送卡片 |
| 次日零点 | 自动清除免打扰标记 |

> **注意**：免打扰标记仅影响 §3 天气触发和 §4 联动触发，用户主动发起叫车（§2）不受影响。

## 出行时间推断逻辑

1. 查询近 30 天工作日出行记录（需用户授权历史数据）
2. 提取每次出行的出发时间，计算众数（精确到 15 分钟粒度）
3. 在「众数时间 - 30 分钟」时调用天气接口检测
4. 历史记录 < 5 条时，降级使用默认窗口 7:30–9:00，在 7:00 检测天气
