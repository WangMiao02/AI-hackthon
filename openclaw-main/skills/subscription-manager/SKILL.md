# subscription_manager — 会员/订阅管理

## 0. Demo 模式声明

本 Skill 运行于 Demo 模式。订阅数据来源于本地 mock 数据库（`references/mock-subscriptions.json`），
模拟用户多平台订阅服务聚合信息。数据覆盖视频、音乐、工具、健身、外卖会员等主流品类。
取消操作模拟成功但不触发真实退订。

**接口模拟方式**：
```
GET /api/subscriptions?user_id=demo_user
Response: references/mock-subscriptions.json

POST /api/subscriptions/cancel
Request: { "sub_id": "xxx" }
Response: { "status": "success", "effective_date": "..." }
```

## 1. 执行入口判断

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时触发（到期提醒） | `trigger_type == "expiry_reminder"` | -> S4 到期提醒推送 |
| 定时触发（月报） | `trigger_type == "monthly_report"` | -> S5 订阅支出月报 |
| 定时触发（僵尸检测） | `trigger_type == "zombie_check"` | -> S6 僵尸订阅检测 |
| 用户主动查询 | 对话中含会员/订阅/续费等意图 | -> S2 前置数据拉取 |
| 用户要求取消 | 对话中含取消/退订意图 | -> S7 取消流程 |
| 用户首次使用 | 无订阅配置记录 | -> S8 Onboarding |

---

## 2. 前置数据拉取

**执行步骤**：

1. 调用订阅聚合接口：`GET /api/subscriptions?user_id=demo_user`
2. 解析返回数据，按状态分组：
   - `expiring_soon`（7天内到期）— 需要决策
   - `active`（正常使用中）— 信息类
   - `auto_renew_on`（自动续费开启中）— 需要用户知悉
   - `expired`（已过期）— 可忽略
3. 计算月度订阅总支出
4. 数据就绪 -> 跳转 S3 展示

**用户响应分支**：

| 操作 | 处理 |
|------|------|
| 继续订阅/帮我续费 | 确认扣费，模拟续费成功 |
| 取消自动续费 | 跳转 S7 取消流程 |
| 不续了 | 标记为不续费，到期后归入已过期 |
| 提醒我 | 到期当天再推送一次 |
| 不管了 | 不做操作，保持原状 |

## 6. 僵尸订阅检测

> 定时触发：每月15号静默检查，发现僵尸订阅才推送。

**判断规则**：
- 过去30天使用次数为 0 → 僵尸订阅
- 过去30天使用次数 < 3 且月费 > 15元 → 疑似僵尸

**推送内容**：
```
💤 发现你有 [N] 个订阅好久没用了：

[服务名称] — ¥[金额]/月
上次使用：[X]天前
本月使用：0次

[服务名称] — ¥[金额]/月
上次使用：[X]天前
本月使用：[X]次

这些每月花费 ¥[合计]，要不要清理一下？

操作：[帮我取消] [先留着] [下月再看]
```

## 7. 取消流程

**执行步骤**：

1. 确认用户要取消的服务
2. 展示取消须知：
   - 剩余有效期（取消后是否立即失效/到期失效）
   - 取消后是否影响其他权益（如联合会员）
   - 是否有退款（年费制是否按比例退）
3. 用户确认 -> 调用取消接口
4. 展示结果

```
确认取消 [服务名称]？

取消须知：
· 取消后仍可使用至 [到期日]
· 不会退还已付费用
· 取消后 [相关影响说明]

确认：[取消订阅] [我再想想]
```

**取消成功**：
```
已取消 [服务名称] 自动续费 ✓
你仍可使用至 [到期日]，之后将自动停止服务。
```

## 8. Onboarding（首次配置）

```
帮你管理所有付费订阅，避免不知不觉被扣费～

你现在用着哪些付费服务？比如：
· 视频：爱奇艺、腾讯视频、B站大会员、Netflix
· 音乐：网易云、QQ音乐、Spotify
· 工具：iCloud、WPS、Notion、ChatGPT Plus
· 生活：美团外卖会员、饿了么会员、京东Plus
· 健身：Keep、超级猩猩月卡
· 其他：Apple One、Steam...

告诉我你在用的，我帮你建档追踪。
也可以说"帮我查"，我试试从消费记录里自动识别。
```

每项需要的信息：

| 配置项 | 示例 |
|--------|------|
| 服务名称 | 爱奇艺黄金VIP |
| 套餐周期 | 月付/季付/年付 |
| 费用 | 25元/月 |
| 到期日 | 2026-06-15 |
| 自动续费 | 已开启（微信支付） |
| 使用频率 | 经常/偶尔/很少 |

## 9. 边界处理速查

| 场景 | 处理 |
|------|------|
| 用户未配置任何订阅 | 触发 Onboarding 引导 |
| 家庭共享类订阅 | 标注"家庭版"，取消前提醒会影响其他家庭成员 |
| 试用期订阅 | 高优提醒：试用到期前1天特别提醒，避免自动转付费 |
| 年费订阅到期 | 到期前7天提醒（比月费提前更久） |
| 联合会员（如88VIP） | 标注包含哪些子服务，取消时说明影响范围 |
| 用户问优惠券/次卡 | 告知"优惠券由领券助手管理，我负责付费订阅" |
| 用户问生活缴费 | 告知"水电物业由缴费管家管理" |
| 海外订阅汇率变动 | 标注当前换算金额，汇率波动超10%时提醒 |
