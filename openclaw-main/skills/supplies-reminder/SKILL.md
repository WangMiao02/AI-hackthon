# supplies_reminder — 日用消耗品补买提醒

## 1. 执行入口判断

**首先判断触发来源，然后跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 月度后台检查 | `trigger_type == "scheduled_check"`（每月1号 09:00） | → §3 库存计算 |
| 定时推送任务 | `trigger_type == "scheduled_push"`（动态日期触发） | → §4 推送提醒卡片 |
| 用户主动唤起 | 对话中含日用品/补货/消耗品等意图 | → §2 前置检查 |
| 用户未完成配置 | 日用品信息缺失 | → §5 Onboarding |

---

## 2. 前置检查（用户主动触发时执行）

1. 读取用户日用品配置信息
2. 如果**配置缺失** → 跳转 §5 Onboarding，Skill 结束
3. 如果**配置完整** → 跳转 §4 推送提醒卡片

---

## 3. 月度后台检查（静默，不推送）

> 仅 `scheduled_check` 触发时执行，用户无感知。

**执行步骤**：

1. 读取用户预填日用品清单（品类、用户自估使用周期、上次购买日期）
2. 读取历史订单辅助校准上次购买日期（预填数据优先）
3. 对每个品类计算预测耗尽日：

   ```
   预测耗尽日 = 上次购买日期 + 用户自估使用周期
   实际推送日 = 预测耗尽日 - 3天
   ```

4. 如果推送日 ≤ 今日 → 立即触发 §3 推送
5. 否则 → 将推送任务写入调度队列，到期自动触发 §3

---

## 4. 推送提醒卡片

### 3.1 卡片输出格式

```
┌──────────────────────────────────────┐
│ 🧴 有 3 件日用品即将用完             │
│                                      │
│ 洗发水                               │
│ 预计 5月14日 用完 · 还剩 2 天        │
│                                      │
│ 卷纸（24卷装）                       │
│ 预计 5月17日 用完 · 还剩 5 天        │
│                                      │
│ 洗衣液                               │
│ 预计 5月19日 用完 · 还剩 7 天        │
│                                      │
│    [帮我下单]      [稍后提醒]        │
└──────────────────────────────────────┘
```

**用户响应分支**：
- 点击 `帮我下单` → 传入日用品清单，触发 `skill_order`，Skill 结束
- 点击 `稍后提醒` → 推迟24小时再次推送，Skill 结束
- 无响应 → 不重复打扰，下次月度检查重新计算

---

## 5. Onboarding（首次配置）

引导用户逐一填写家庭日用品信息：

| 配置项 | 示例 |
|--------|------|
| 品类名称 | 洗发水 |
| 用户自估使用周期 | 45天 |
| 上次购买日期 | 2026-04-01 |
| 月度检查日（默认每月1号） | 每月1号 |

> 已配置用户可在「家庭设置」中增删品类或修改月度检查日。
> 每次购买后建议更新上次购买日期，或通过美团历史订单自动同步（需授权）。

---

## 6. 边界处理速查

| 场景 | 处理 |
|------|------|
| 用户未填写日用品信息 | 触发 Onboarding 引导，Skill 暂不执行 |
| 库存充足（本月内不会耗尽） | 本月不推送，下次检查日重新计算 |
| 推送日已过（库存严重不足） | 跳过调度队列，立即推送 |
| 用户点击「稍后提醒」 | 推迟24小时重推，最多推迟2次 |

---

## 7. 本期不做（Backlog）

> 以下功能**不在当前 Skill 范围内**，遇到请告知用户暂不支持：

- 食品生鲜类补货提醒
- 家庭成员分别管理各自用品
- 基于实际用量的智能周期修正（现阶段完全依赖用户自估）
- 跨平台历史订单同步（仅美团）
