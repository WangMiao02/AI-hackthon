# meituan_review_manager — 商家评价管理

## 1. 执行入口判断

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 新评价事件 | 美团 API 推送新评价通知 | → §2 评价分流 |
| 每日汇总写入 | `trigger_type == "daily_summary"`（营业结束后触发） | → §5 日报写入 |
| 配置缺失 | 首次使用或配置不完整 | → §6 Onboarding |

> **核心设计原则**：Skill 由新评价事件驱动，无需商家主动唤起。
> 好评全自动处理，差评推卡片等待商家确认，商家仅需处理差评部分。

---

## 2. 评价分流

收到新评价后，按星级进入对应处理路径：

```
1-2星（差评）  →  §3 差评处理
3-5星（好评）  →  §4 好评处理
```

---

## 3. 差评处理

### 3.1 AI 生成回复草稿

读取评价内容，结合以下维度生成回复草稿：

- 客户投诉点（口味 / 份量 / 配送 / 包装 / 其他）
- 商家已配置的品牌语气（见 §6 Onboarding）
- 历史差评回复风格（若有）

### 3.2 推送差评处理卡片

```
┌──────────────────────────────────────┐
│ 🔴 新差评  ⭐☆☆☆☆  刚刚             │
│                                      │
│ 「配送太慢了，等了一个小时，         │
│   而且汤都洒了，很失望」             │
│                                      │
│ ────────────────────────────────── │
│ 💬 AI 建议回复                       │
│                                      │
│ 您好，非常抱歉给您带来了不好的体验。 │
│ 配送时效和包装问题我们已记录，会立   │
│ 即跟进改进。欢迎您再次光临，我们一   │
│ 定给您更好的体验！                   │
│                                      │
│  [直接发送]  [修改后发送]  [暂不回复]│
│                                      │
│  [疑似违规，发起申诉 →]              │
└──────────────────────────────────────┘
```

**按钮响应**：
- `直接发送` → 调用 API 发送草稿内容，推送发送成功卡片
- `修改后发送` → 展开文本编辑区，商家修改后点击「确认发送」
- `暂不回复` → 关闭卡片，计入「待处理差评」，可在每日汇总卡片中查看
- `疑似违规，发起申诉 →` → 跳转 `meituan_review_appeal`（待开发），传入当前评价内容

### 3.3 修改后发送交互

## 4. 好评处理

### 4.1 自动生成并发送回复

读取评价内容，从商家预设好评回复模板库中匹配最合适的模板，
自动调用 API 发送，全程无需商家介入。

> 模板匹配维度：提及菜品名 / 提及配送 / 纯泛好评 / 含晒图
> 同一用户不重复使用相同模板，避免机械感。

### 4.2 好评回复不单独推送通知

好评回复静默完成，仅在每日汇总卡片中呈现当日好评回复总数。

---

## 5. 日报写入

每日营业结束后，将以下汇总数据写入 `merchant_daily_report`：

| 数据项 | 说明 |
|--------|------|
| 当日新增差评数 | 1-2星评价总数 |
| 差评已回复数 | 商家确认发送的回复数 |
| 差评待处理数 | 点击「暂不回复」或未响应的差评数 |
| 当日新增好评数 | 3-5星评价总数 |
| 好评自动回复数 | 系统自动发送的好评回复数 |

> 写入格式与口径以 `merchant_daily_report` §3 数据项定义为准。

## 7. 边界处理速查

| 场景 | 处理方式 |
|------|---------|
| API 读取评价失败 | 推送失败提示卡片，附 [手动前往美团商家版] 按钮 |
| API 发送回复失败 | 自动重试 1 次；仍失败则推送失败卡片，草稿内容保留可重新发送 |
| 差评商家长时间未响应（超过24小时） | 推送「有差评待处理」提醒卡片，附差评列表 |
| 好评模板全部用尽（轮转完毕） | 从头轮转，同一用户不连续重复同一模板 |
| 商家修改回复内容为空 | 不允许发送，提示「回复内容不能为空」 |
