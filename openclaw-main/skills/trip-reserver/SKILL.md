# trip_reserver — 出行提醒与预订

## 1. 执行入口判断

**首先判断触发来源，然后跳转对应分支：**

| 来源 | 判断方式 | 跳转 |
|------|---------|------|
| 定时系统触发 | `trigger_type == "scheduled"` | → §3 查票流程 |
| 用户主动唤起 | 用户对话中包含出行/订票意图 | → §2 前置检查 |
| 用户无固定行程 | 用户首次使用 | → §5 Onboarding |

---

## 2. 前置检查（用户主动触发时执行）

1. 读取用户已配置的固定行程列表
2. 如果**无配置** → 跳转 §5 Onboarding，Skill 结束
3. 如果**有配置** → 确认用户意图指向哪条行程，跳转 §3

---

## 3. 查票与推荐

**输入**：用户行程配置（出发地、目的地、出行日期、偏好）

**执行步骤**：
1. 并行调用火车票 + 机票实时接口，查询目标日期票次
2. 按以下规则各取最优 **1 条**：

   | 优先级 | 筛选维度 |
   |--------|---------|
   | 1 | 用户历史惯常时段（早/中/晚班匹配） |
   | 2 | 偏好座位有余票 |
   | 3 | 同等条件取价格最低 |

3. 如用户偏好单一交通方式，只推对应类型
4. 如推荐票次**已售罄** → 展示次优票次，并附文字说明

**跳转** → §4 推送提醒卡片

---

## 4. 推送与确认预订

### 4.1 出行提醒卡片（输出格式）

```
┌──────────────────────────────────────┐
│ ✈️ 周三 北京 → 上海，还有 3 天出发   │
│                                      │
│ 推荐票次                             │
│ 🚄 高铁  G1次  07:00→11:30  ¥553    │
│ ✈️ 飞机  CA1234  08:00→09:30  ¥820  │
│                                      │
│ [选高铁]   [选飞机]   [自己看看]     │
└──────────────────────────────────────┘
```

**用户响应分支**：
- 点击`选高铁` / `选飞机` → 展示 §4.2 详情确认页
- 点击`自己看看` → 跳转美团搜索结果页，Skill 结束

### 4.2 票次详情确认页（对话内展示）

```
┌──────────────────────────────────────┐
│ 🚄 G1次  北京南 → 上海虹桥           │
│ 出发：周三 07:00   到达：11:30       │
│ 座位：二等座 靠窗                    │
│ 票价：¥553.00                        │
│ 支付方式：美团支付（默认）           │
│                                      │
│      [确认预订]     [换一张]         │
└──────────────────────────────────────┘
```

**用户响应分支**：
- 点击`确认预订` → 调用预订接口 + 支付 → §4.3
- 点击`换一张` → 返回 §3 重新查票

### 4.3 预订结果处理

**成功**：
```
✅ 预订成功！
G1次  北京南 → 上海虹桥
周三 07:00 出发 · 二等座靠窗 · ¥553.00
📌 出发前一天晚上我会再提醒你。
```
→ 设置出发前一天 + 当天早上的二次提醒任务

**失败**：
→ 提示失败原因，保留当前预订信息，引导用户重试或换票

---

## 5. Onboarding（首次配置固定行程）

引导用户依次填写：

| 配置项 | 示例 |
|--------|------|
| 出发地 | 北京 |
| 目的地 | 上海 |
| 出行频率 | 每周三 |
| 出行偏好 | 均可 / 高铁 / 飞机 |
| 座位偏好 | 靠窗 / 靠走道 / 无偏好 |

配置完成后自动跳转 §3 查票流程。

> 已配置用户可在「行程管理」页新增 / 删除 / 暂停固定行程。

---

## 6. 边界处理速查

| 场景 | 处理 |
|------|------|
| 未配置行程 | 触发 §5 Onboarding |
| 推荐票已售罄 | 展示次优票，注明原因 |
| 同一行程已预订 | 不重复推送，标注"已预订" |
| 支付失败 | 提示原因，保留信息可重试 |
| 行程取消 | 用户在行程管理中删除/暂停 |

---

## 7. 本期不做（Backlog）

> 以下功能**不在当前 Skill 范围内**，遇到请告知用户暂不支持：

- 临时行程（日历读取 / 口头告知）
- 退改签
- 境外 / 多程中转
- 酒店联动推荐
- 航班实时延误推送
