# Skill：一键领券
**Skill Name**：`coupon_claim`  
**所属Agent**：`business_travel_agent`  
**版本**：v0.1（Hackathon阶段）  
**作者**：sxn  
**更新时间**：2026-05-17

---

## 1. Skill 定位

根据用户历史消费习惯，每日定时推送个性化可领券卡片，用户一键全领，领取后附带有效期提醒，帮助商务差旅用户在消费前自动获取最优惠券，无需手动找券。

**本期不做**：第三方平台券、跨平台券聚合。仅覆盖美团生态内券种。

---

## 2. 触发条件

### 2.1 主动触发（定时推送）
- 触发时机：每日上午 10:00
- 触发条件：当日存在用户**尚未领取的可用券**，且与用户历史消费习惯匹配

### 2.2 被动触发（用户主动唤起）
- 用户在对话界面主动发起，如"帮我看看有什么券可以领"
- 用户在10:00之前主动打开App，可通过按钮手动触发

---

## 3. 前置依赖

### 3.1 数据依赖
- 用户历史消费记录（美团内部，需用户授权）：用于过滤券种、计算消费关联性
- 美团券池数据（实时读取）：获取当前可领券列表及力度信息

### 3.2 Onboarding 配置（可选）
用户可在设置中手动排除不感兴趣的券种（如从不点外卖则屏蔽外卖券），未配置时默认由历史消费数据自动过滤。

---

## 4. 券的筛选与排序规则

### 4.1 筛选逻辑
- 过滤掉与用户历史消费**无关联**的类目券（如用户从不使用丽人服务，则过滤丽人券）
- 过滤掉用户**已领取未使用**的同类券（避免重复推送）

### 4.2 排序逻辑
按以下两个维度综合倒序排列（力度权重 > 关联性权重）：

| 维度 | 说明 |
|------|------|
| 券力度 | 优惠金额 / 折扣力度，绝对金额优先 |
| 消费关联性 | 与用户近期高频消费类目的匹配程度 |

示例：打车满50减15 > 酒旅9.5折 > 外卖满30减5（假设用户高频打车+酒旅，偶尔点外卖）

### 4.3 展示数量
- 卡片内默认展示前3张
- 超过3张时折叠，用户可点击展开查看全部

---

## 5. 执行流程

```
[定时触发 10:00 or 用户主动唤起]
        ↓
读取用户历史消费记录 + 当日可领券池
        ↓
按筛选规则过滤，按排序规则排列
        ↓
是否存在匹配券？
   ├─ 否 → 推送「今日暂无匹配券」提示，结束
   └─ 是 → 推送领券卡片（展示前3张，超出折叠）
                ↓
        用户点击「一键全领」
                ↓
        批量调用领券接口
                ↓
        领取成功 → 推送有效期提醒
        领取失败（部分/全部）→ 告知失败原因，可重试
```

## 6. 输出规格

### 6.1 领券卡片（推送样式）

```
┌──────────────────────────────────────┐
│ 🎟 今日有 5 张券等你领               │
│                                      │
│ 🥇 打车券   满50减15                 │
│ 🥈 酒旅券   9.5折                    │
│ 🥉 外卖券   满30减5                  │
│    + 2张更多优惠 ∨                   │
│                                      │
│          [一键全领]                  │
└──────────────────────────────────────┘
```

### 6.2 领取成功后的有效期提醒

```
✅ 已成功领取 5 张券！

· 打车券   满50减15   有效期至 5月24日
· 酒旅券   9.5折       有效期至 5月31日
· 外卖券   满30减5    有效期至 5月20日
· ...

📌 外卖券将于 3 天后到期，记得使用！
```

> 临近到期提醒规则：有效期 ≤ 3天的券，在有效期提醒中高亮标出。

### 6.3 领取失败提示

```
⚠️ 本次领取部分失败：
· 打车券 已达领取上限
· 其余 4 张已成功领取

[重试失败项]
```

---

## 7. 边界与限制

| 场景 | 处理方式 |
|------|----------|
| 当日无匹配券 | 推送友好提示，不打扰用户 |
| 用户已领取同类券未使用 | 过滤掉，不重复推送 |
| 券领取接口调用失败 | 区分部分失败/全部失败，提示可重试 |
| 用户屏蔽了某类券 | 严格遵守设置，不推送对应类目 |
| 展开折叠后发现券已被抢完 | 实时标注"已抢完"，不可领取 |

---

## 8. 本期不做（Backlog）

- 单张券独立领取 / 取消某张不领
- 券到期前独立推送提醒（本期仅在领取时附带提醒）
- 第三方平台券聚合
- 券使用场景主动推荐（如用户打车时主动提示"你有打车券可用"）
