# Skill：一键报销
**Skill Name**：`travel_reimbursement`  
**所属Agent**：`business_travel_agent`  
**版本**：v0.1（Hackathon阶段）  
**作者**：sxn  
**更新时间**：2026-05-17

---

## 1. Skill 定位

帮助商务差旅用户自动识别可报销消费记录，定时推送报销提醒，用户一键确认后生成标准报销单（App内图片 + 可导出PDF），减少手动整理发票的时间成本。

**本期不做**：对接用户公司报销系统自动提交（B层及以上）。报销单生成后由用户自行处理后续流程。

---

## 2. 触发条件

### 2.1 主动触发（定时推送）
- 触发频率：每天 or 每周（用户在Onboarding时选择）
- 推送时机：工作日上午 9:00
- 触发条件：用户账户内存在**上一个统计周期内尚未报销的可报销消费记录**

### 2.2 被动触发（用户主动唤起）
- 用户在对话界面主动发起，如"帮我看看有哪些可以报销"
- 用户点击报销卡片上的「一键开具」按钮

---

## 3. 前置依赖

### 3.1 Onboarding 配置（首次使用必填）
用户需在首次使用时完成以下配置，Skill 才能正常运行：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 可报销消费类目 | 用户勾选哪些类型的美团消费可报销 | ✅ 酒旅 ✅ 打车 ❌ 外卖 |
| 推送频率 | 每天 or 每周（每周几） | 每周五 |
| 报销单抬头 | 生成报销单时使用的姓名/工号 | 张思怡 / EMP-20241 |

> 配置入口：商务差旅Agent首页 → 「报销设置」

### 3.2 数据依赖
- 美团内部交易数据（实时读取，需用户授权）
- 消费类目标签（由美团侧提供，与Onboarding配置做交叉过滤）

---

## 4. 执行流程

```
[定时触发 or 用户主动唤起]
        ↓
读取用户近期消费记录
        ↓
按Onboarding配置过滤可报销类目
        ↓
是否存在未报销记录？
   ├─ 否 → 推送「暂无可报销记录」提示，结束
   └─ 是 → 推送报销卡片（含消费明细列表，默认全选）
                ↓
        用户可手动取消勾选条目 → 合计金额实时更新
                ↓
        用户点击「一键开具」
                ↓
        是否有选中条目？
           ├─ 否 → 提示「未选择任何条目」，返回卡片
           └─ 是 → 生成报销单图片（App内预览）
                        ↓
                用户选择：「完成」 or 「导出PDF」
```

---

## 5. 输出规格

### 5.1 报销卡片（推送样式）
卡片内直接展示消费明细，无需跳转，默认全选，用户可取消勾选，合计金额实时响应。

```
┌─────────────────────────────────────┐
│ 💼 本周有 3 条消费记录可报销          │
│ 统计周期：5月10日 - 5月16日          │
│                                      │
│ ✅ 5月12日  打车   ¥  89.00          │
│ ✅ 5月13日  酒旅   ¥ 980.00          │
│ ✅ 5月15日  打车   ¥ 211.00          │
│                      合计：¥1,280.00 │
│                                      │
│      [一键开具]      [稍后提醒]      │
└─────────────────────────────────────┘
```

### 5.2 消费明细条目字段
每条记录展示：
- 勾选框（默认全选）
- 消费日期
- 消费类目（酒旅 / 打车）
- 消费金额
- 订单号（折叠展示，用于财务核查）

### 5.3 报销单内容
| 字段 | 来源 |
|------|------|
| 报销人姓名/工号 | Onboarding配置 |
| 报销日期 | 生成当日 |
| 消费明细列表 | 用户确认后的条目 |
| 合计金额 | 自动汇总 |
| 底部备注 | "由真探AI管家生成" |

输出格式：App内图片预览 + 一键导出PDF

---

## 6. 边界与限制

| 场景 | 处理方式 |
|------|----------|
| 用户未完成Onboarding配置 | 触发引导流程，Skill暂不执行 |
| 统计周期内无可报销记录 | 推送友好提示，不打扰用户 |
| 用户手动取消勾选所有条目 | 不生成报销单，提示"未选择任何条目" |
| 同一条消费记录已报销过 | 过滤掉，不重复推送 |
| 消费记录数据读取失败 | 提示"数据获取失败，请稍后重试" |
