# Skill：订酒店
**Skill Name**：`hotel_reserver`  
**所属Agent**：`business_travel_agent`  
**版本**：v0.1（Hackathon阶段）  
**作者**：sxn  
**更新时间**：2026-05-17

---

## 1. Skill 定位

用户通过 `trip_reserver` 完成交通票预订后，立即触发酒店推荐卡片，根据用户偏好推荐目的地办公地址附近的酒店，用户在对话内直接完成预订。

**本期不做**：独立触发的酒店搜索、多目的地酒店推荐、退订流程。

---

## 2. 触发条件

### 2.1 主动触发（联动触发）
- 触发时机：`trip_reserver` 预订成功后**立即**触发
- 触发条件：用户未开启「我有公司协议酒店」开关

### 2.2 被动触发（用户主动唤起）
- 用户在对话界面主动发起，如"帮我在上海订个酒店"
- 用户在行程管理页手动触发酒店推荐

---

## 3. 前置依赖

### 3.1 Onboarding 配置

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 常去办公地址 | 目的地城市的惯常办公地点，作为距离参照点 | 上海市静安区南京西路1515号 |
| 价位偏好 | 单晚预算区间 | ¥300-600 |
| 房型偏好 | 大床 / 双床 / 无偏好 | 大床 |
| 公司协议酒店 | 开启后不推送酒店推荐 | 开 / 关（默认关） |

> 配置入口一：Onboarding 引导流程  
> 配置入口二：Agent首页 → 「行程管理」→ 差旅偏好设置

### 3.2 数据依赖
- 美团酒店实时数据（房价、库存、距离，需接入）
- 用户Onboarding配置（价位、房型、办公地址）
- `trip_reserver` 预订结果（目的地城市、出行日期）

---

## 4. 酒店推荐规则

默认展示2条推荐，超出折叠，排序规则如下：

| 维度 | 说明 |
|------|------|
| 距离 | 距用户常去办公地址步行/驾车距离，优先近的 |
| 价位匹配度 | 优先落在用户Onboarding填写的预算区间内 |
| 房型可用性 | 优先有用户偏好房型的酒店 |
| 评分 | 同等条件下优先评分较高的酒店 |

---

## 5. 执行流程

```
[trip_reserver 预订成功，立即触发]
        ↓
检查用户是否开启「公司协议酒店」开关
   ├─ 是 → 不推送，结束
   └─ 否 → 读取用户Onboarding偏好配置
                ↓
        查询目的地酒店实时数据
                ↓
        按推荐规则筛选排序，取前2条展示
                ↓
        是否有匹配酒店？
           ├─ 否 → 提示「暂无符合偏好的酒店」，引导用户自选
           └─ 是 → 推送酒店推荐卡片（默认展示2条，超出折叠）
                        ↓
                用户选择酒店 → 展示酒店详情确认页
                        ↓
                用户点击「确认预订」
                        ↓
                调用预订接口，在对话内完成支付
                        ↓
                预订成功 → 推送预订确认信息
                预订失败 → 提示失败原因，可重试或换一家
```

## 7. 边界与限制

| 场景 | 处理方式 |
|------|----------|
| 用户开启公司协议酒店开关 | 不触发推送，静默跳过 |
| 推荐酒店均无偏好房型 | 推送时注明"暂无大床房，已推荐其他房型" |
| 推荐酒店价格超出预算 | 推送时注明超出预算幅度，由用户决策 |
| 用户点击「自己看看」 | 跳转至美团酒店目的地搜索结果页 |
| 支付失败 | 提示失败原因，保留预订信息可重试 |
| 所选酒店已满房 | 提示已满房，推荐次优选项 |
