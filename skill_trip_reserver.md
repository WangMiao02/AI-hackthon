# Skill：出行提醒与预订
**Skill Name**：`trip_reserver`  
**所属Agent**：`business_travel_agent`  
**版本**：v0.1（Hackathon阶段）  
**作者**：sxn  
**更新时间**：2026-05-17

---

## 1. Skill 定位

根据用户预设的固定出行行程，提前三天主动推送出行提醒，附带推荐票次（飞机+高铁），用户在对话内直接完成预订，全程无需跳转。

**本期不做**：临时行程识别（日历读取）、境外出行、退改签。

---

## 2. 触发条件

### 2.1 主动触发（定时推送）
- 触发时机：出行日期前三天，当日上午 9:00
- 触发条件：用户存在已配置的固定行程，且该行程尚未完成预订

### 2.2 被动触发（用户主动唤起）
- 用户在对话界面主动发起，如"帮我看看下周三的票"
- 用户在行程管理页手动触发查询

---

## 3. 前置依赖

### 3.1 Onboarding 配置（核心入口）
用户首次使用时填写固定行程信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 出发地 | 固定出发城市 | 北京 |
| 目的地 | 固定目的地城市 | 上海 |
| 出行频率 | 每周几 / 每月几号 | 每周三 |
| 出行偏好 | 高铁 / 飞机 / 均可 | 均可 |
| 座位偏好 | 靠窗 / 靠走道 / 无偏好 | 靠窗 |

> 配置入口一：Onboarding 引导流程  
> 配置入口二：Agent首页 → 「行程管理」→ 添加固定行程（适用于Agent上线后新增行程）

### 3.2 数据依赖
- 美团火车票实时票务数据（需接入）
- 美团机票实时票务数据（需接入）
- 用户历史出行记录（辅助判断偏好，需用户授权）

---

## 4. 票次推荐规则

每次推送附带 1-2 条推荐票次，飞机和高铁各取最优一条（若用户偏好单一交通方式则只推对应类型）：

| 维度 | 说明 |
|------|------|
| 时间匹配度 | 优先推符合用户历史出行时段的票次（如用户惯常早班） |
| 座位可用性 | 优先推有用户偏好座位的票次 |
| 价格 | 同等条件下优先推价格较低的票次 |

---

## 5. 执行流程

```
[出行日前三天 9:00 定时触发 or 用户主动唤起]
        ↓
读取用户固定行程配置
        ↓
是否存在即将到来的行程？
   ├─ 否 → 不推送，结束
   └─ 是 → 查询火车票 + 机票实时数据
                ↓
        按推荐规则各取最优票次（最多各1条）
                ↓
        推送出行提醒卡片（附带推荐票次）
                ↓
        用户选择票次 → 展示票次详情确认页
                ↓
        用户点击「确认预订」
                ↓
        调用预订接口，在对话内完成支付
                ↓
        预订成功 → 出发前一天 + 当天早上再次提醒
        预订失败 → 提示失败原因，建议重新选择票次
```

## 7. 边界与限制

| 场景 | 处理方式 |
|------|----------|
| 用户未配置固定行程 | 触发Onboarding引导，Skill暂不执行 |
| 推荐票次已售罄 | 提示已售罄，展示次优票次或引导用户自选 |
| 用户点击「自己看看」 | 跳转至美团火车票/机票搜索结果页 |
| 支付失败 | 提示失败原因，保留预订信息可重试 |
| 行程临时取消 | 用户可在行程管理中删除或暂停该固定行程 |
| 同一行程已预订 | 不重复推送，标注"已预订"状态 |

---

## 8. 本期不做（Backlog）

- 临时行程识别（日历读取 / 用户口头告知）
- 退改签流程
- 境外出行 / 多程中转
- 酒店联动推荐（出行确认后顺带推目的地酒店）
- 出行当天实时提醒（如航班延误推送）
